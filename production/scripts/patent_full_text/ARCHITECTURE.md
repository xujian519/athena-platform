## 版本信息
- **当前版本**: v3.0.0
- **最后更新**: 2025-12-25
- **状态**: ✅ Phase 1-6 全部完成

---

## 一、设计理念

**专利全文处理系统作为Athena平台的核心基础设施，为专利检索、分析、代理业务提供完整的PDF下载、向量化、知识图谱构建能力。**

### 核心价值
- 🎯 **业务驱动**: 服务于复审、无效、对比文件分析等专利代理业务
- 🚀 **性能优先**: 多级缓存、批处理、异步处理
- 🔧 **可扩展性**: 模块化设计，易于集成新功能
- 📊 **数据智能**: 三层向量化 + 知识图谱深度语义理解

## 三、核心组件详解

### 3.1 检索与输入层

#### patent-search MCP
- **位置**: `/mcp-servers/patent-search-mcp-server/`
- **功能**: 搜索中国专利数据库 (PostgreSQL, 2800万+条记录)
- **返回**: 24字段完整元数据 + PostgreSQL UUID

#### IntegratedPatentDownloader
- **位置**: `/production/dev/scripts/patent_full_text/integrated_downloader.py`
- **功能**: 统一多种输入源，维护元数据映射关系
- **支持**: patent-search结果、手动输入、文件导入

#### PDF监控服务
- **位置**: `/production/dev/scripts/patent_full_text/phase3/pdf_monitor_service.py`
- **功能**: 监控目录变化，自动触发处理流程
- **特性**: 增量检测、状态持久化、回调机制

### 3.2 核心处理管道

#### 向量化处理V2 (VectorProcessorV2)

**三层架构设计**:

```
Layer 1: 全局检索层 (粗筛)
├── 标题向量 (1个)
├── 摘要向量 (1个)
└── IPC分类向量 (1个)

Layer 2: 核心内容层 (精排)
├── 独立权利要求向量 (1-3个)
└── 从属权利要求向量 (5-20个)

Layer 3: 发明内容层 (深度分析)
├── 技术问题向量 (1个)
├── 技术方案向量 (1-3个)
├── 有益效果向量 (1个)
└── 具体实施方式向量 (2-10个)
```

**平均每件专利**: 20-30个向量

#### 三元组提取 (RuleExtractor)

**问题-特征-效果三元组**:
```python
Triple(
    subject="technical_feature",     # 技术特征
    relation="SOLVES",               # 解决
    object="technical_problem"       # 技术问题
)

Triple(
    subject="technical_feature",     # 技术特征
    relation="ACHIEVES",             # 实现
    object="technical_effect"        # 技术效果
)
```

**特征间关系**:
- COMBINATIONAL (组合)
- DEPENDENT (依赖)
- ALTERNATIVE (替代)
- SEQUENTIAL (顺序)
- HIERARCHICAL (层次)
- CAUSAL (因果)

#### 知识图谱构建V2 (PatentKGBuilderV2)

**核心顶点类型**:
- `patent`: 专利基本信息
- `claim`: 权利要求
- `technical_problem`: 技术问题
- `technical_feature`: 技术特征
- `technical_effect`: 技术效果

**核心关系类型**:
- `SOLVES`: 特征→问题
- `ACHIEVES`: 特征→效果
- `RELATES_TO`: 特征→特征

### 3.3 系统优化层

#### 缓存管理器 (CacheManager)
- **MemoryCache**: LRU淘汰、TTL过期、线程安全
- **Redis集成**: 可选的分布式缓存
- **装饰器支持**: `@cached(ttl=3600)`

#### 批处理器 (BatchProcessor)
- 多线程并发处理
- 进度追踪和错误收集
- 性能统计 (成功率、耗时)

#### 重试处理器 (RetryHandler)
- 指数退避策略
- 可配置重试次数
- 异常类型过滤

#### 性能监控器 (PerformanceMonitor)
- 指标记录 (min/max/avg/total)
- 装饰器支持
- 全局单例

### 3.4 数据库集成层

#### QdrantManager
- **集合**: `patent_full_text_v2`
- **向量维度**: 768 (BGE-base-zh-v1.5)
- **距离度量**: Cosine
- **功能**: 连接管理、集合初始化、向量插入、向量搜索、健康检查

#### NebulaManager
- **空间**: `patent_full_text_v2`
- **功能**: 连接池管理、空间初始化、NGQL执行、健康检查

#### DatabaseIntegration
- **统一接口**: 管理所有数据库连接
- **批量初始化**: 一键初始化所有数据库
- **健康检查**: 统一的健康检查接口

### 3.5 部署层

#### Docker Compose
- **Qdrant**: `qdrant/qdrant:v1.12.0`
- **NebulaGraph**: `vesoft/nebula-*:v3.8.0`
- **Redis**: `redis:7-alpine`
- **应用容器**: 自定义镜像

#### DeploymentManager
- 前置条件检查
- 服务部署/停止/启动/重启
- 服务状态查询
- 日志查看
- 健康检查

#### ProductionConfig
- 环境变量加载
- 配置验证
- 多环境支持 (production/development/staging)

## 十、关键设计原则

1. **解耦设计**: 检索、下载、处理、存储相互独立
2. **统一接口**: IntegratedPatentDownloader统一处理多种输入源
3. **自动处理**: PDF监控服务自动触发处理流程
4. **数据关联**: PDF与PostgreSQL记录双向关联 (UUID)
5. **业务驱动**: 所有功能服务于专利代理业务
6. **性能优先**: 多级缓存、批处理、异步处理
7. **可扩展性**: 模块化设计，易于集成新功能
