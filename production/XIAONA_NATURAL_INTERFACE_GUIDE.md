## 🌟 什么是自然语言交互？

小娜自然语言交互系统让您可以**直接用中文对话**来使用提示词系统，无需编写任何代码。

### 传统方式 vs 自然语言方式

**传统方式**（需要编程）:
```python
from xiaona_agent import XiaonaAgent
agent = XiaonaAgent()
response = agent.query("帮我分析技术交底书", "patent_writing")
```

**自然语言方式**（直接对话）:
```
您: 帮我分析这个技术交底书
小娜: 【小娜】好的，我来帮您分析...
```

## 💡 使用技巧

### 技巧1: 描述需求而不是任务名

**❌ 不太好的说法**:
```
您: 执行task_1_1
```

**✅ 更好的说法**:
```
您: 帮我分析这个技术交底书，找出核心创新点
```

### 技巧2: 提供完整背景信息

**❌ 不太好的说法**:
```
您: 怎么答复？
```

**✅ 更好的说法**:
```
您: 审查员说我的权利要求1不具备创造性（A22.3），
引用了对比文件D1和D2，我该怎么答复？
```

### 技巧3: 可以粘贴实际内容

```
您: 帮我分析这个技术交底书：

技术领域：智能物流
发明名称：一种基于深度学习的智能分拣系统

技术问题：
现有物流分拣效率低、错误率高

技术方案：
1. 使用YOLOv5进行目标检测
2. 使用DeepSORT进行跟踪
3. 使用强化学习优化分拣路径

技术效果：
分拣效率提升40%，错误率降低至0.1%

小娜: 【小娜】好的，我来帮您分析这个技术交底书。

📊 核心创新点识别：
├── 技术领域: 智能物流分拣
├── 技术问题: 效率低、错误率高
├── 技术方案: 深度学习+强化学习组合
└── 技术效果: 效率提升40%，错误率降至0.1%

🤝 需要您确认：
1. 是否需要检索现有技术？
2. 是否需要进一步分析各技术特征的创新性？
...
```

### 技巧4: 指定模式控制上下文使用

**快速问答（节省上下文）**:
```
您: 用最简模式快速检查，这个权利要求清楚吗？
```

**标准任务（推荐）**:
```
您: 帮我分析这个审查意见
```

**复杂任务（详细指导）**:
```
您: 用完整模式帮我从头到尾撰写这个专利申请文件
```

### 技巧5: 多轮对话

```
您: 帮我分析这个技术交底书
小娜: 【小娜】好的，我发现了3个创新点...

您: 第一个创新点有相关现有技术吗？
小娜: 【小娜】我来帮您检索关于"深度学习目标检测"的现有技术...

您: 如果审查员认为这个没有创造性，我该怎么争辩？
小娜: 【小娜】如果审查员引用类似技术，您可以这样争辩...
```

## 🆘 故障排查

### 问题1: 小娜理解错了我的意思

**解决方案**: 更具体地描述您的需求

```
❌ 您: 分析一下
✅ 您: 帮我分析这个技术交底书的核心创新点
```

### 问题2: 上下文空间不足

**解决方案**: 压缩对话历史或使用更小的模式

```
您: 压缩对话历史
小娜: 【小娜】对话历史已压缩，现在有更多空间了
```

或者：

```
您: 用最简模式继续
```

### 问题3: 想要退出对话

**解决方案**: 使用退出指令

```
您: 退出
或
您: quit
或
您: q
或按 Ctrl+C
```
