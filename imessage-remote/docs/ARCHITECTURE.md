## 核心组件

### 1. iMessage RPC Client (iMessage 客户端)

**职责**：
- 与 iMessage 通信（通过 imsg 命令）
- 发送和接收消息
- 处理 RPC 请求和响应

**关键方法**：
```python
async def start()              # 启动客户端
async def stop()               # 停止客户端
async def send_message()       # 发送消息
def on_message(callback)       # 注册消息回调
```

### 2. Command Parser (命令解析器)

**职责**：
- 解析自然语言命令
- 识别目标智能体
- 提取任务类型和参数

**解析流程**：
```
原始命令
    ↓
检测智能体（小诺/Athena）
    ↓
检测任务类型（专利检索/分析/查询）
    ↓
提取参数（关键词、专利号等）
    ↓
生成 ParsedCommand 对象
```

### 3. Command Router (命令路由器)

**职责**：
- 路由命令到对应智能体
- 管理任务执行状态
- 收集执行结果

**路由规则**：
- 默认路由到小诺
- 明确指定时路由到 Athena
- 根据任务类型动态选择

### 4. Agents (智能体)

#### Xiaonuo Agent（小诺）
- **专利检索**：关键词搜索、过滤
- **专利分析**：创造性评估、对比分析
- **信息查询**：联系人信息
- **提醒事项**：创建提醒

#### Athena Agent（雅典娜）
- **复杂分析**：深度推理、综合分析
- **专利深度分析**：法律风险、对比专利
- **报告生成**：多维度分析报告

### 5. Obsidian Writer (Obsidian 写入器)

**职责**：
- 格式化结果为 Markdown
- 按任务类型组织文件
- 写入 Obsidian 仓库

**文件组织**：
```
Athena/
├── 📊 专利检索/
│   └── 2026-01-31/
│       └── 人工智能专利检索.md
├── 📈 专利分析/
│   └── 2026-01-31/
│       └── 创造性分析_CN202310123456.md
└── 🧠 Athena对话/
    └── 2026-01-31/
        └── 深度分析_技术方案.md
```

### 6. Smart Feedback Generator (智能反馈生成器)

**职责**：
- 根据任务复杂度生成反馈
- 自动调整详细程度

**反馈模式**：
- **Simple**（<30s）：状态 + 链接
- **Standard**（30-120s）：状态 + 关键数据 + 链接
- **Detailed**（>120s）：完整摘要 + 发现 + 链接 + 建议
- **Smart**：自动选择

## 数据流

### 命令执行流程

```
1. 用户发送 iMessage
   ↓
2. IMessageClient 接收消息
   ↓
3. Security Layer 验证发件人
   ↓
4. CommandParser 解析命令
   ↓
5. CommandRouter 路由到智能体
   ↓
6. Agent 执行任务
   ↓
7. ObsidianWriter 写入详情
   ↓
8. FeedbackGenerator 生成摘要
   ↓
9. IMessageClient 发送反馈
   ↓
10. 用户接收反馈
```

### 错误处理流程

```
异常发生
    ↓
记录日志
    ↓
生成错误反馈
    ↓
发送给用户
    ↓
（可选）写入错误日志到 Obsidian
```

## 并发模型

- **异步处理**：所有 I/O 操作都是异步的
- **任务队列**：支持并发执行多个任务
- **超时控制**：每个任务都有超时限制
- **错误隔离**：单个任务失败不影响其他任务

## 安全机制

1. **白名单**：只允许可信发件人
2. **命令验证**：验证命令格式和参数
3. **超时保护**：防止任务无限期运行
4. **日志审计**：记录所有操作

## 扩展性

### 添加新的智能体

```python
class CustomAgent(BaseAgent):
    async def execute(self, command: ParsedCommand) -> Dict[str, Any]:
        # 实现自定义逻辑
        pass
```

### 添加新的任务类型

在 `config/config.yaml` 中添加：

```yaml
command_patterns:
  custom_task:
    patterns:
      - "自定义任务.*"
    agent: "xiaonuo"
    task_type: "custom_task"
```

### 自定义反馈格式

继承 `SmartFeedbackGenerator` 并重写生成方法。

## 性能考虑

- **连接池**：复用 HTTP 连接
- **缓存**：缓存频繁查询的结果
- **批处理**：批量写入 Obsidian
- **异步日志**：避免阻塞主流程
