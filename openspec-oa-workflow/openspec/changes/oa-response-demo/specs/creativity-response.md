# 创造性答复规范

## 场景描述

收到审查意见：对比文件1+对比文件2的结合，权利要求1不具备创造性。

## 输入要求

### 必需输入
- 审查意见通知书（含对比文件信息）
- 原始权利要求书
- 原始说明书

### 可选输入
- 对比文件1全文
- 对比文件2全文
- 前次答复（如有）

---

## 答复流程

### Step 1: 理解审查意见

系统 **SHALL** 执行：
1. 提取审查员认定的最接近现有技术（D1）
2. 提取审查员认定的区别特征
3. 提取审查员引用的对比文件2（D2）
4. 理解审查员的结合理由

### Step 2: 分析技术方案

系统 **SHALL** 执行：
1. 理解本申请的技术方案
2. 识别权利要求1的全部技术特征
3. 分析每个特征的技术效果
4. 确定发明点

### Step 3: 构建答复策略

系统 **SHOULD** 选择以下策略之一或组合：

| 策略 | 适用条件 | 论证要点 |
|------|----------|----------|
| A. 区别特征未被公开 | D2未公开区别特征 | 逐特征对比分析 |
| B. 无结合启示 | D1+D2结合无动机 | 技术领域/问题/效果差异 |
| C. 预料不到的技术效果 | 结合后效果超预期 | 协同效应/显著提升 |
| D. 技术教导相反 | D2教导相反方向 | 引用D2原文佐证 |

### Step 4: 撰写答复

系统 **SHALL** 按以下结构输出：

```markdown
## 关于权利要求1的创造性

审查员认为：[复述审查意见]

申请人对上述意见持有不同看法，理由如下：

### 一、区别技术特征的认定

本申请权利要求1与对比文件1相比，存在以下区别技术特征：

[列出区别特征]

上述区别技术特征带来了以下技术效果：

[说明技术效果]

因此，权利要求1实际解决的技术问题是：[确定技术问题]

### 二、对比文件2未公开区别技术特征 / 未给出结合启示

[论证内容]

### 三、本申请具备创造性

基于上述分析，本领域技术人员在面对所述技术问题时，
没有动机将对比文件1和对比文件2结合，从而得到权利要求1
的技术方案。因此，权利要求1具备突出的实质性特点。

同时，权利要求1的技术方案带来了[技术效果]，该效果
是[预料不到的/显著的]。因此，权利要求1具备显著的进步。

综上所述，权利要求1具备专利法第22条第3款规定的创造性。
```

## 质量检查清单

### 必检项

- [ ] 三步法逻辑完整
- [ ] 每个区别特征都有分析
- [ ] 技术效果有依据
- [ ] 法条引用正确
- [ ] 无自相矛盾

### 建议项

- [ ] 引用对比文件原文
- [ ] 使用审查员熟悉的表述
- [ ] 篇幅适中（不过长不过短）
- [ ] 结论明确
