## ❌ 失败教训与解决方案

### 1. 模块导入问题

#### ❌ 问题: Python模块导入失败
```python
# 失败代码
from knowledge_enhanced_patent_analyzer import KnowledgeEnhancedPatentAnalyzer
# ModuleNotFoundError: No module named 'knowledge_enhanced_patent_analyzer'
```

**原因分析**:
- 缺少`__init__.py`文件使目录成为Python包
- 相对导入路径设置错误
- PYTHONPATH环境变量配置不当

**✅ 解决方案**:
```python
# 1. 添加__init__.py文件
touch /path/to/perception/__init__.py

# 2. 修正导入路径
from .knowledge_enhanced_patent_analyzer import KnowledgeEnhancedPatentAnalyzer

# 3. 环境变量配置
export PYTHONPATH="/project/src:$PYTHONPATH"
```

**教训**: 在复杂Python项目中，模块结构设计需要在开发初期就规划好。

### 2. API服务部署问题

#### ❌ 问题: API服务频繁重启
```bash
# 错误的启动方式
python3 simple_patent_api.py &  # 服务启动后立即退出

# 错误信息
WARNING: You must pass the application as an import string to enable 'reload'
```

**原因分析**:
- 直接执行Python文件与uvicorn期望的模块导入不兼容
- 后台进程管理不当
- 缺少正确的进程守护机制

**✅ 解决方案**:
```python
# 正确的启动方式
# 方法1: 使用uvicorn命令
uvicorn simple_patent_api:app --host=0.0.0.0 --port=8000

# 方法2: Python内嵌启动
import uvicorn
from simple_patent_api import app
uvicorn.run(app, host='0.0.0.0', port=8000, log_level='info')

# 方法3: 生产环境部署
nohup uvicorn simple_patent_api:app --host=0.0.0.0 --port=8000 > api.log 2>&1 &
```

**教训**: API服务部署需要理解框架的启动机制，不能简单地将脚本放在后台运行。

### 3. 依赖管理问题

#### ❌ 问题: 依赖包版本冲突
```bash
# 错误信息
ImportError: cannot import name 'TechnicalFeatureExtractor'
ModuleNotFoundError: No module named 'Union'
```

**原因分析**:
- 不同模块使用了不同版本的依赖
- 缺少完整的requirements.txt文件
- 系统Python与虚拟环境混用

**✅ 解决方案**:
```python
# 1. 完整的依赖管理
pip freeze > requirements.txt

# 2. 类型导入修正
from typing import Dict, List, Any, Optional, Union  # 添加Union

# 3. 统一开发环境
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**教训**: 复杂项目必须严格管理依赖，使用虚拟环境隔离。

### 4. 测试策略问题

#### ❌ 问题: API测试502错误
```python
# 测试代码
response = requests.get("http://localhost:8000/api/v1/health")
# 返回: HTTP Error 502: Bad Gateway
```

**原因分析**:
- API服务进程异常退出
- 端口冲突或绑定问题
- 请求超时设置不当

**✅ 解决方案**:
```python
# 1. 服务状态检查
def check_service_health():
    try:
        response = requests.get("http://localhost:8000/", timeout=5)
        return response.status_code == 200
    except requests.exceptions.RequestException:
        return False

# 2. 渐进式测试
# 先测试根路径，再测试具体端点
# 先测试简单GET，再测试复杂POST

# 3. 使用TestClient进行集成测试
from fastapi.testclient import TestClient
client = TestClient(app)
response = client.get("/api/v1/health")
```

**教训**: API测试需要验证服务状态，不能假设服务始终可用。
