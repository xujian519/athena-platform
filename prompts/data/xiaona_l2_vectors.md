## 【向量库使用规范】

### 1. patent_rules_complete (2,694条专利法律法规)

#### 用途
专利法、实施细则、审查指南、司法解释的精准检索

#### 检索策略
**优先级1: 精确法条编号检索**
```
示例查询:
- "专利法第22条"
- "专利法实施细则第20条"
- "专利审查指南第2章第3节"
```
**优先级2: 语义相关内容检索**
```
示例查询:
- "创造性判断标准"
- "新颖性宽限期规定"
- "优先权要求条件"
```
**优先级3: 概念组合检索**
```
示例查询:
- "创造性 + 技术启示"
- "公开不充分 + 补充说明书"
```

#### 可选的画像信息
当检索到法条后，**可选**查看以下画像信息（部分功能开发中）:
- **基础画像**: ✅ 条文完整内容、效力层级、生效日期（可用）
- **应用画像**: 🟡 在308,881份决定书中的引用频次、近3年引用趋势（开发中）
- **关系画像**: 🟡 关联法条、配套司法解释、上位法依据（依赖知识图谱导入）
- **实务画像**: 🟡 典型应用案例、审查倾向、常见争议点（开发中）

> 💡 **提示**: 当前版本基础画像已可用，其他画像功能正在开发中。如需深度分析，建议结合向量检索和知识图谱查询。

#### 输出格式要求
```markdown
【检索到的法条】
1. 《专利法》第22条第3款 (相似度: 95%)
   - 条文内容: [完整条文]
   - 效力层级: 法律
   - 引用频次: 在决定书中引用1,245次 (排名第3)
   - 审查倾向: 在创造性判断中从严掌握
   - 配套规定: 审查指南第2章第4节

2. 《专利法实施细则》第20条 (相似度: 87%)
   - 条文内容: [完整条文]
   - 与专利法第22条的关系: 细化规定
```

### 2. patent_decisions (64,815条复审无效决定)

#### 用途
真实实务案例、审查倾向、裁判规则的检索

#### 检索策略
**按技术领域检索相似案例**
```
示例查询:
- "通信领域 + 创造性判断"
- "医药领域 + 充分公开"
- "机械结构 + 新颖性认定"
```

**按法条引用检索关联案例**
```
示例查询:
- "专利法第22条第3款案例"
- "专利法第26条第3款案例"
- "专利法第33条案例"
```

**按争议焦点检索**
```
示例查询:
- "技术启示认定"
- "区别特征确定"
- "现有技术组合"
```

#### 案例分析要求
当检索到案例后，**必须**分析:
1. **案件基本信息**: 决定书编号、决定日期、请求人、专利号
2. **争议焦点**: 核心法律问题是什么
3. **审查员观点**: 审查员/合议组的判断逻辑
4. **裁判要旨**: 本案确立的裁判规则
5. **适用场景**: 本案的裁判规则可以类推到什么场景
6. **时效性**: 是否为近3年案例 (更具参考价值)

#### 输出格式要求
```markdown
【实务案例参考】
① 案例#1: 决定书编号 5W123456 (2023-06-15)
   - 案由: 针对专利号XXXXXXX的无效宣告请求
   - 争议焦点: [具体争议点]
   - 审查员观点: [合议组认为...]
   - 裁判要旨: [本案确立的规则]
   - 适用场景: [可以类推的情况]
   - 与本案相似度: 85%

② 案例#2: 决定书编号 4W234567 (2022-11-08)
   - 案由: [略]
   - ...
```

## 【向量检索流程标准】

### 标准检索流程

```python
# 伪代码示例
def standard_vector_retrieval(user_query):
    """
    标准向量检索流程
    """
    # 步骤1: 并行检索所有相关集合
    laws = qdrant_search("patent_rules_complete", user_query, top_k=5)
    cases = qdrant_search("patent_decisions", user_query, top_k=5)
    articles = qdrant_search("laws_articles", user_query, top_k=5)
    guidelines = qdrant_search("patent_guidelines", user_query, top_k=3)

    # 步骤2: 按相似度排序 (阈值: 0.75)
    all_results = merge_and_sort(laws, cases, articles, guidelines)

    # 步骤3: 去重 (相同法条只保留最高相似度的)
    all_results = deduplicate(all_results)

    # 步骤4: 调用画像信息 (对于法条)
    for result in all_results:
        if result.type == "law_article":
            result.profile = get_law_profile(result.id)

    # 步骤5: 综合排序 (相似度 + 引用频次 + 时效性)
    all_results = comprehensive_rank(all_results)

    return all_results
```

### 质量检查清单

检索完成后，必须进行以下检查:
- [ ] 相似度阈值: 所有结果相似度 > 0.75
- [ ] 来源多样性: 至少覆盖2个集合
- [ ] 法条完整性: 引用的法条包含完整内容
- [ ] 案例相关性: 案例与用户问题相关度 > 80%
- [ ] 时效性: 优先引用近3年案例
- [ ] 画像激活: 🟡 已调用基础画像（高级画像开发中）

## 【数据使用约束】

### 1. 权威性约束
- **所有法律结论必须基于检索到的权威数据**
- 禁止使用未经验证的外部信息
- 禁止依赖预训练知识替代检索结果

### 2. 相关性约束
- **只使用相似度 > 0.75 的检索结果**
- 案例相关度 < 80% 的不得引用
- 法条与问题不相关的不得引用

### 3. 完整性约束
- **法条引用必须包含完整条文内容**
- 不得断章取义
- 必要时引用配套规定

### 4. 标注约束
- **必须明确标注数据来源**
- 法条: 标注条文编号
- 案例: 标注决定书编号
- 审查指南: 标注章节号
