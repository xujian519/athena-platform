## 【搜索触发条件】

### ✅ 必须搜索的场景

1. **创造性分析前** → 背景技术调研
2. **答复审查意见前** → 技术理解 + 对比文件分析
3. **专利撰写前** → 技术现状 + 术语规范
4. **遇到不懂的技术术语** → 概念理解

### ❌ 不需要搜索的场景

1. 纯法律条文问题 → 用向量库
2. 案例检索 → 用向量库
3. 专利摘要检索 → 用数据库

---

## 【标准搜索流程】

```
确定目标 → 执行搜索 → 筛选验证 → 整合输出
```

### 步骤1: 确定目标
- 明确需要了解什么
- 确定关键词
- 选择引擎

### 步骤2: 执行搜索
- Tavily优先
- Bocha中文补充
- Metaso深度搜索
- Google Patents专利专用

### 步骤3: 筛选验证
- 相关性>80%
- 权威性验证
- 时效性检查
- 多源交叉验证

### 步骤4: 整合输出
- 提取关键信息
- 结构化内容
- 标注来源和时间
- 指出局限性

## 【质量标准】

**相关性**: 搜索结果与问题相关性>80%

**权威性**: 优先权威来源(学术/官方/行业)

**时效性**: 技术信息优先近1-3年

**完整性**: 多源交叉验证,明确局限性

---

## 【使用约束】

**必须**:
- 只在需要时搜索
- 先内后外
- 多源验证
- 标注来源

**禁止**:
- 过度依赖搜索
- 不验证质量
- 不标注来源
