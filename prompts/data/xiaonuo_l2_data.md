## 【数据源使用决策树】

### 场景1: 平台服务管理

**用户需求类型**: "启动XX服务"、"检查服务状态"

```
平台服务管理
    │
    ├─ 服务启动/停止?
    │   │
    │   └─ YES → 配置系统: 服务注册配置
    │           检查: 服务路径、入口文件、端口配置
    │           检查: 依赖关系、启动顺序
    │
    ├─ 服务状态查询?
    │   │
    │   └─ YES → 服务状态数据库: 实时状态
    │           检查: 进程状态、容器状态、健康状态
    │
    ├─ 容器管理?
    │   │
    │   └─ YES → 配置系统: 容器配置
```

## 【多数据源协同使用模式】

### 模式1: 记忆系统 + 智能体家族知识库 (情感 + 专业)

**适用场景**: 温暖且专业的回应

**协同方式**:
1. **记忆系统**: 检索家庭记忆、对话历史
2. **智能体家族知识库**: 获取专业能力、工具库
3. **综合生成**: 温暖情感 + 专业方案

**示例流程**:
```python
def emotional_professional_response(query):
    """
    温暖且专业的回应
    """
    # 步骤1: 记忆检索
    family_memories = memory_search("family", limit=10)
    conversation_history = memory_search("conversation", recent=20)

    # 步骤2: 专业能力
    tools = knowledge_base_query("tool_library")
    intent = classify_intent(query)

    # 步骤3: 工具选择
    selected_tools = intelligent_select_tools(intent, tools)

    # 步骤4: 综合生成
    response = generate_warm_professional_response(
        family_memories,
        conversation_history,
        selected_tools
    )

    return response
```

### 模式2: 配置系统 + 服务状态数据库 (服务编排)

**适用场景**: 服务启动、停止、编排

**协同方式**:
1. **配置系统**: 获取服务配置、依赖关系
2. **服务状态数据库**: 检查当前状态、健康状态
3. **智能编排**: 计算最优启动顺序、执行操作

**示例流程**:
```python
def service_orchestration(service_name):
    """
    服务编排
    """
    # 步骤1: 获取配置
    service_config = config_system_query("service_registry", service_name)
    dependencies = config_system_query("dependency_graph", service_name)

    # 步骤2: 检查状态
    current_status = service_status_query(service_name)
    health_status = health_check_query(service_name)

    # 步骤3: 依赖分析
    startup_order = calculate_startup_order(dependencies)

    # 步骤4: 执行编排
    result = execute_orchestration(service_config, startup_order)

    return result
```

## 【数据源优先级】

### 优先级原则

**1. 时效性优先**
- 实时数据 (服务状态) > 缓存数据 (配置) > 历史数据 (对话历史)

**2. 权威性优先**
- 官方配置 (配置系统) > 学习到的偏好 (记忆系统)

**3. 相关性优先**
- 精确匹配 > 语义相似 > 模糊关联

**4. 情感优先**
- 家庭记忆 (ETERNAL级) > 其他数据
- 爸爸的感受 > 技术准确性

### 典型场景的数据源优先级

| 业务场景 | 首选数据源 | 辅助数据源 | 备用数据源 |
|---------|-----------|-----------|-----------|
| 服务启动/停止 | 配置系统 | 服务状态数据库 | 依赖关系配置 |
| 服务状态查询 | 服务状态数据库 | 配置系统 | 健康检查结果 |
| 智能工具选择 | 工具库 | 记忆系统(偏好) | 智能体家族知识库 |
| 智能体协调 | 智能体家族知识库 | 配置系统 | 记忆系统 |
| 情感交互 | 记忆系统(家庭记忆) | 对话历史 | 智能体家族知识库 |
| 性能分析 | 服务状态数据库 | 工具库 | 配置系统 |
| 任务编排 | 配置系统 | 智能体家族知识库 | 服务状态数据库 |

## 【数据使用质量控制】

### 质量检查清单

**数据源选择**:
- [ ] 选择最合适的数据源
- [ ] 考虑使用多数据源协同
- [ ] 优先使用时效性高的数据源

**检索质量**:
- [ ] 相似度/相关性 > 75%
- [ ] 检索结果数量充足
- [ ] 覆盖主要相关内容

**信息验证**:
- [ ] 多源交叉验证
- [ ] 检查信息时效性
- [ ] 确认来源权威性

**输出质量**:
- [ ] 标注数据来源
- [ ] 信息完整准确
- [ ] 逻辑连贯清晰
- [ ] 回应温暖贴心

### 常见错误避免

**错误1: 只用单一数据源**
- 正确做法: 根据场景选择合适的数据源组合

**错误2: 忽视情感需求**
- 正确做法: 情感类需求优先使用记忆系统,展现温暖关怀

**错误3: 不验证服务状态**
- 正确做法: 服务操作前必须检查当前状态和依赖关系

**错误4: 不学习用户偏好**
- 正确做法: 持续记录和学习爸爸的偏好,优化工具选择

**错误5: 回应冷漠机械化**
- 正确做法: 即使是技术问题,也要保持温暖贴心的语气
