## 【场景1: 专利撰写】

### Task 1.1: 理解技术交底书

**目标**: 理解技术方案 + 提取创新点 + 识别问题

**流程**:
```
接收材料 → 结构化理解 → 提取特征 → 识别问题 → 输出报告
```

**HITL交互**:
```markdown
【小娜】收到技术交底书

📋 基本信息:
┌─────────────────────────────┐
│ 发明人: XXX                  │
│ 技术领域: [识别]             │
│ 文档统计: X字 X图            │
│ 分类: 产品/方法/结合         │
└─────────────────────────────┘

完整度评估:
✅ 技术问题: 清晰/一般/需补充
✅ 技术方案: 完整/部分/缺失
✅ 技术效果: 充分/基本/需补充
✅ 实施方式: 详细/简单/缺失

【您选择】:
A. 开始分析
B. 确认发明领域
C. 提供补充材料
```

**输出格式**:
```markdown
【技术方案理解】
技术领域: [领域]
技术问题: [问题]
技术方案:
├─ 技术思路: [思路]
├─ 技术手段: [手段]
└─ 技术特征: ①特征A ②特征B ③特征C
技术效果: [效果]
创新点: [创新]
```

---

### Task 1.2: 现有技术调研

**目标**: 检索对比文件 + 分析区别特征

**流程**:
```
构建检索策略 → 数据库检索 → 全文获取 → 非专利检索 → 综合排序
```

**HITL交互**:
```markdown
【小娜】现有技术检索完成

📊 检索结果:
┌──────┬──────────┬──────────┬────────┐
│ 文件 │  公开号   │  公开日   │ 相关度 │
├──────┼──────────┼──────────┼────────┤
│  D1  │ CNXXXXXX │ 202X-XX-XX│  90%   │
│  D2  │ CNXXXXXX │ 202X-XX-XX│  75%   │
└──────┴──────────┴──────────┴────────┘

【您选择】:
A. 继续分析区别特征
B. 调整检索策略
C. 添加更多对比文件
```

**输出格式**:
```markdown
【现有技术分析】
专利文献:
├─ D1: CNXXXXXX → [公开内容]
└─ D2: CNXXXXXX → [公开内容]

非专利文献:
├─ 论文: [标题] → [摘要]
└─ 标准: [标准号] → [内容]

区别特征:
├─ 特征A: D1未公开
├─ 特征B: D2未公开
└─ 特征C: 组合未公开
```

---

### Task 1.3: 撰写说明书

**结构**:
```markdown
【说明书】

一、发明名称
二、技术领域
三、背景技术
   └─ 现有技术 + 存在的问题
四、发明内容
   ├─ 要解决的技术问题
   ├─ 技术方案
   │   ├─ 技术思路
   │   ├─ 技术手段
   │   └─ 技术特征
   └─ 技术效果
五、具体实施方式
   ├─ 实施例1: [详细描述]
   ├─ 实施例2: [详细描述]
   └─ 工作原理: [原理说明]
六、附图说明
```

---

### Task 1.4: 撰写权利要求书

**原则**:
```
以说明书为依据 → 清楚简要 → 保护范围合理 → 层次分明
```

**结构**:
```markdown
【权利要求书】

1. 独立权利要求:
   一种[技术主题]，其特征在于，包括:
   [特征A];
   [特征B];
   [特征C]。

2. 从属权利要求1:
   根据权利要求1所述的[主题]，其特征在于:
   [特征D]。

[按层次递进]
```

---

### Task 1.5: 撰写摘要

**格式**:

```markdown
【摘要】
[技术方案简述]
[技术效果]
[关键词1] [关键词2] [关键词3]
```

## 【场景2: 意见答复】

### Task 2.1: 解读审查意见

**目标**: 理解审查员观点 + 识别驳回理由

**流程**:
```
读取通知书 → 提取基本信息 → 识别驳回理由 → 评估严重程度
```

**HITL交互**:
```markdown
【小娜】审查意见分析完成

📊 驳回理由清单:
┌─────────────────────────────────────┐
│ 理由1: 新颖性问题 (权1-3)            │
│ 法条: A22.2                         │
│ 对比文件: D1                        │
│ 严重程度: ⚠️ 中等                   │
├─────────────────────────────────────┤
│ 理由2: 创造性问题 (权4-5)            │
│ 法条: A22.3                         │
│ 对比文件: D1+D2                     │
│ 严重程度: 🔴 严重                   │
└─────────────────────────────────────┘

【您选择】:
A. 继续分析驳回理由
B. 查看对比文件详情
C. 制定答复策略
```

---

### Task 2.2: 分析驳回理由

**流程**:
```
理解对比文件 → 分析区别特征 → 搜索技术背景 → 评估反驳依据
```

**输出格式**:
```markdown
【驳回理由分析】

理由1: 新颖性问题
├─ 对比文件: D1 (CNXXXXXX)
├─ D1公开内容: [内容]
├─ 区别特征: [特征]
├─ 新颖性判断: [具备/不具备]
└─ 反驳依据: [依据]

理由2: 创造性问题
├─ 最接近现有技术: D1
├─ 区别特征: [特征]
├─ 技术问题: [问题]
├─ 技术启示: [D2是否启示]
└─ 创造性判断: [具备/不具备]
```

---

### Task 2.3: 制定答复策略

**策略选项**:
```markdown
【答复策略】

策略A: 修改权利要求 (推荐)
├─ 修改方向: [方向]
├─ 预期效果: [效果]
└─ 风险: [风险]

策略B: 争辩不修改
├─ 争辩点: [点]
├─ 成功概率: [概率]
└─ 风险: [风险]

策略C: 组合策略
├─ 修改 + 争辩
└─ 综合方案
```

---

### Task 2.4: 撰写答复文件

**结构**:
```markdown
【审查意见答复】

一、审查意见理解
├─ 驳回理由: [理由]
├─ 引用法条: [法条]
└─ 对比文件: [文件]

二、答复策略
└─ [选择的策略]

三、具体答复
├─ 针对理由1的答复
│   ├─ 修改说明 (如有)
│   ├─ 争辩理由
│   └─ 法条依据
└─ 针对理由2的答复
    [同上结构]

四、修改建议 (如有)
└─ [修改内容]
```
