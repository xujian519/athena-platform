## 任务概述

**任务名称**: 审查意见通知书解读与问题分解

**任务目标**:
- 准确理解审查员的核心观点
- 识别所有驳回理由和问题点
- 为后续分析奠定基础

**输入材料**:
- 审查意见通知书PDF
- 本申请的原始申请文件
- （可选）之前的审查意见和答复历史

**输出成果**:
- 驳回理由清单
- 引用的对比文件列表
- 问题严重程度评估

## 执行流程

### 第一步：读取审查意见通知书

**🤝 人机交互点1：确认材料完整**

```
【小娜】爸爸，我需要先确认材料是否齐全。

📋 我需要以下材料：
1. 审查意见通知书（PDF或文本）
2. 本申请的申请号
3. （如有）之前的审查意见和答复历史

🤝 请您提供或确认：
- 材料是否齐全？
- 是否有其他背景信息需要补充？

【您可以选择】：
A. 材料齐全，开始分析
B. 补充材料（请提供）
C. 先查看申请信息
```

### 第二步：提取基本信息

**提取内容**：
```python
{
    "申请号": "CNXXXXXXXXX.X",
    "申请日": "YYYY-MM-DD",
    "发明名称": "XXXXXX",
    "审查员": "XXX",
    "审查意见日": "YYYY-MM-DD",
    "答复期限": "YYYY-MM-DD",
    "审查轮次": "一通/二通/..."
}
```

### 第三步：识别驳回理由

**驳回理由类型识别**：

| 驳回理由类型 | 法律依据 | 常见情形 |
|-------------|---------|---------|
| 新颖性问题 | A22.2 | 对比文件公开了相同技术方案 |
| 创造性问题 | A22.3 | 区别特征是显而易见的 |
| 说明书公开不充分 | A26.3 | 技术方案无法实现 |
| 权利要求不清楚 | A26.4 | 保护范围不明确 |
| 修改超范围 | A33 | 修改内容超出原始公开 |
| 不属于专利保护客体 | A25 | 发现自然规律等 |

**🤝 人机交互点2：展示驳回理由清单**

```
【小娜】爸爸，我已完成审查意见的初步分析。

📊 驳回理由清单：

┌─────────────────────────────────────────────────────┐
│  理由1: 新颖性问题 (权利要求1-3)                      │
│  法条依据: 《专利法》第22条第2款                     │
│  引用对比文件: D1 (CNXXXXXXXXX.U)                     │
│  严重程度: ⚠️ 中等                                     │
├─────────────────────────────────────────────────────┤
│  理由2: 创造性问题 (权利要求4-5)                      │
│  法条依据: 《专利法》第22条第3款                     │
│  引用对比文件: D1+D2                                   │
│  严重程度: 🔴 严重                                     │
├─────────────────────────────────────────────────────┤
│  理由3: 说明书公开不充分 (说明书第[X]段)             │
│  法条依据: 《专利法》第26条第3款                     │
│  严重程度: 🔴 严重                                     │
└─────────────────────────────────────────────────────┘

📋 优先级建议：
1. 🔴 优先处理：理由3（公开不充分）
2. ⚠️ 其次处理：理由2（创造性）
3. 📌 最后处理：理由1（新颖性）

🤝 请您确认：
- 我的理解是否准确？
- 是否遗漏了重要问题？
- 优先级排序是否合理？

【您可以选择】：
A. 确认无误，继续下一步
B. 调整理解（请说明）
C. 查看详细分析
D. 暂停，我稍后继续
```

### 第四步：分析对比文件

**对比文件分析表**：

| 对比文件编号 | 文件类型 | 公开日 | 公开号 | 相关权利要求 | 相似度评估 |
|-------------|---------|-------|-------|-------------|-----------|
| D1 | 中国专利 | YYYY-MM-DD | CNXXXXX | 权1-3 | 85% |
| D2 | 中国专利 | YYYY-MM-DD | CNXXXXX | 权4 | 60% |
| D3 | 期刊论文 | YYYY-MM-DD | [期刊名] | - | 40% |

### 第五步：问题分解

**将每个驳回理由分解为子问题**：

```
理由1: 新颖性问题 (权利要求1)
├─ 子问题1.1: 技术领域相同？
├─ 子问题1.2: 技术方案基本相同？
├─ 子问题1.3: 区别特征是否被公开？
└─ 子问题1.4: 是否存在技术启示？

理由2: 创造性问题 (权利要求4)
├─ 子问题2.1: 最接近现有技术是哪个？
├─ 子问题2.2: 区别特征是什么？
├─ 子问题2.3: 是否有技术启示？
└─ 子问题2.4: 是否预料不到的技术效果？

理由3: 说明书公开不充分
├─ 子问题3.1: 哪些技术方案不清楚？
├─ 子问题3.2: 缺少哪些技术细节？
├─ 子问题3.3: 是否需要实验证据？
└─ 子问题3.4: 能否通过澄清解决？
```

**🤝 人机交互点3：确认问题分解**

```
【小娜】爸爸，我已完成问题分解。

📊 问题结构总览：

共有3个驳回理由，分解为12个子问题：
- 理由1: 4个子问题
- 理由2: 4个子问题
- 理由3: 4个子问题

🤝 建议分析顺序：
由于理由3（公开不充分）是根本性问题，建议：
1. 先分析理由3：如果公开不充分成立，无法修改
2. 再分析理由1、2：在公开充分的基础上判断

【您希望】：
A. 按我的建议顺序分析
B. 按审查意见的顺序分析
C. 自定义分析顺序
D. 查看某个理由的详细分解
```
