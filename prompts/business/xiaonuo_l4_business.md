## 【场景1: 服务启动管理】

### 场景描述
管理Athena平台服务的启动,包括单服务启动、批量启动、依赖处理和启动验证。

### 标准执行流程

```
┌─────────────────────────────────────────────────────────┐
│  步骤1: 理解需求                                         │
│  - 目标服务: [服务名]                                    │
│  - 启动模式: 单个/批量                                   │
│  - 特殊要求: [如有]                                      │
│  ↓                                                       │
│  步骤2: 配置检查                                         │
│  - 服务配置存在性                                        │
│  - 入口文件检查                                          │
│  - 端口配置检查                                          │
│  ↓                                                       │
│  步骤3: 依赖分析                                         │
│  - 识别依赖服务                                          │
│  - 检查依赖状态                                          │
│  - 计算启动顺序                                          │
│  ↓                                                       │
│  步骤4: 执行启动                                         │
│  - 按顺序启动服务                                        │
│  - 记录PID/容器ID                                        │
│  - 验证启动成功                                          │
│  ↓                                                       │
│  步骤5: 结果报告                                         │
│  - 启动成功/失败                                         │
│  - 服务地址信息                                          │
│  - 后续建议                                              │
└─────────────────────────────────────────────────────────┘
```

### 输入参数规范

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| service_name | string | 是 | 服务名称 |
| startup_mode | enum | 否 | 启动模式: single/batch/all |
| check_health | boolean | 否 | 是否执行健康检查,默认true |
| wait_for_ready | boolean | 否 | 是否等待服务就绪,默认true |

### 输出格式规范

```json
{
  "success": true,
  "service": "nlp-service",
  "pid": 12345,
  "port": 8001,
  "health_status": "healthy",
  "startup_time": "2025-12-26T12:00:00",
  "message": "服务启动成功",
  "endpoints": {
    "health": "http://localhost:8001/health",
    "api": "http://localhost:8001/api"
  }
}
```
