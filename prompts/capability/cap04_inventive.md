## 执行流程: "三步法"

### 第一步: 确定最接近的现有技术

#### 1.1 检索现有技术

```python
# 伪代码示例
def find_closest_prior_art(target_patent):
    """查找最接近的现有技术"""

    # 从patent_db检索相同或相近技术领域的专利
    prior_arts = query_postgresql("""
        SELECT patent_number, application_date, title, abstract, ipc_main_class
        FROM patents
        WHERE ipc_main_class = %s  -- 相同IPC主分类
          AND application_date < %s  -- 早于目标申请日
        ORDER BY application_date DESC
        LIMIT 50
    """, (target_patent["ipc_main_class"], target_patent["application_date"]))

    # 向量检索语义相似的专利
    similar_patents = qdrant_search(
        "patent_full_text",  # 专利全文向量库
        target_patent["title"] + " " + target_patent["abstract"],
        top_k=20
    )

    # 合并并排序
    candidates = merge_and_rank(prior_arts, similar_patents)

    return candidates
```

#### 1.2 选择最接近的现有技术

**选择标准**:
1. **技术领域相同或相近**: IPC分类相同或相近
2. **技术问题相同或相近**: 解决类似的技术问题
3. **技术效果相同或相近**: 达到类似的技术效果
4. **技术特征重合度高**: 公开了最多的技术特征
5. **申请日早于目标专利**: 必须早于目标专利的申请日

**输出格式**:

### 第二步: 确定区别特征和实际解决的技术问题

#### 2.1 识别区别特征

**对比分析**:
```python
def identify_distinguishing_features(target, prior_art):
    """识别区别特征"""

    # 提取目标专利的技术特征
    target_features = extract_features(target["claims"])
    # 提取现有技术的技术特征
    prior_art_features = extract_features(prior_art["description"])

    # 找出区别特征
    distinguishing = target_features - prior_art_features

    return distinguishing
```

**输出格式**:
```markdown

[具体描述技术问题]

- D1存在的问题: [D1的不足或缺陷]
- 目标专利的改进: [如何解决D1的问题]
- 技术效果提升: [具体提升的技术效果]
```

#### 2.2 分析技术效果

**技术效果类型**:
1. **性能提升**: 速度、精度、效率等提升
2. **成本降低**: 生产成本、使用成本降低
3. **结构简化**: 结构更简单、体积更小
4. **功能增强**: 功能更多、应用更广
5. **可靠性提高**: 稳定性、耐用性提高

**输出格式**:

### 第三步: 判断是否显而易见

#### 3.1 技术启示检索

```python
def check_technical_teaching(distinguishing_features, prior_arts):
    """检查是否存在技术启示"""

    # 检索其他现有技术是否公开了区别特征
    teachings = []
    for feature in distinguishing_features:
        # 向量检索
        results = qdrant_search("patent_full_text", feature, top_k=10)
        # 知识图谱查询
        graph_results = query_knowledge_graph(feature)
        # 合并结果
        teachings.append({
            "feature": feature,
            "found_in_prior_art": results,
            "teaching_exists": len(results) > 0
        })

    return teachings
```

#### 3.2 技术启示判断标准

**审查指南规定**:

《专利审查指南》第2章第4.2节规定:

"要求保护的发明相对于现有技术是显而易见的，是指所属技术领域的技术人员在现有技术的基础上，仅仅通过合乎逻辑的分析、推理或者有限的试验就可以得到要求保护的技术方案。"

**判断要素**:
1. **区别特征是否已被公开**: 其他现有技术是否公开了区别特征
2. **是否存在结合启示**: 是否有动机将区别特征应用到最接近的现有技术
3. **结合是否容易**: 是否仅需简单组合或常规手段
4. **是否产生预料不到的效果**: 是否有预料不到的技术效果

**输出格式**:
```markdown

1. 区别特征D是否被其他现有技术公开?

- D3 (论文XXX, 2021-08-15): 提到了特征D的概念

分析:
D2公开了与特征D类似的技术手段，但应用场景不同。

结论: 区别特征D未被其他现有技术明确公开。

2. 是否存在将区别特征D应用于D1的技术启示?

4. 是否有预料不到的技术效果?

- 效果1: [具体效果]，超出预期[具体说明]
- 效果2: [具体效果]，超出预期[具体说明]

类似案例参考:

结论: [存在/不存在]预料不到的技术效果。
```
