| **Tavily** | AI增强搜索 | ✅ | 综合技术搜索 | ✅ 主要 |
| **Bocha (博查)** | 中文AI搜索 | ✅ | 中文技术资料 | ✅ 辅助 |

| **Google Patents** | 专利搜索 | ✅ | 国际专利检索 | ✅ 专用 |

## 【搜索引擎使用规范】

### 1. 何时使用搜索引擎

在专利业务中，**以下场景必须使用搜索引擎进行技术调研**:

#### 场景1: 创造性分析前的背景技术调研

**触发条件**: 当用户需要进行新颖性/创造性分析时

**搜索目标**:
- 理解专利涉及的技术领域基础知识
- 了解相关技术的发展历程和技术现状
- 识别技术领域的常用技术手段
- 查找可能被遗漏的对比文件

**搜索策略**:
```python
# 伪代码示例
def background_technology_search(patent_info):
    """
    创造性分析前的背景技术调研
    """
    queries = [
        f"{patent_info['技术领域']} 基础知识",
        f"{patent_info['核心特征']} 工作原理",
        f"{patent_info['技术问题']} 现有解决方案",
        f"{patent_info['技术领域']} 最新进展"
    ]

    # 使用Tavily进行综合搜索
    results = search_manager.search(
        query=queries,
        engines=[SearchEngineType.TAVILY],
        max_results=10
    )

    return summarize_background(results)
```

**输出要求**:
```markdown

基础知识:
├─ 核心概念: [技术术语解释]
├─ 工作原理: [技术原理说明]
├─ 常用实现: [该领域的常见技术手段]
└─ 发展历程: [技术演进过程]

技术现状:
├─ 主流方案: [当前主流技术路线]
├─ 技术瓶颈: [现有技术存在的问题]
├─ 最新进展: [近3年技术发展]
└─ 技术趋势: [未来发展方向]

数据来源: Tavily AI搜索 (检索时间: YYYY-MM-DD)
```

#### 场景2: 答复审查意见前的技术理解

**触发条件**: 当收到审查意见驳回时

**搜索目标**:
- 深入理解审查员引用的对比文件
- 查找反驳审查意见的技术依据
- 搜索支持修改方向的技术资料
- 了解审查员可能的技术认知局限

**搜索策略**:
```python
def office_action_response_search(office_action):
    """
    答复审查意见前的技术搜索
    """
    # 步骤1: 理解对比文件技术
    citation_queries = [
        f"{office_action['citation']} 工作原理",
        f"{office_action['citation']} 技术细节",
        f"{office_action['citation']} 与{申请技术} 区别"
    ]

    # 步骤2: 搜索技术差异证据
    difference_queries = [
        f"{申请特征} 技术优势",
        f"{申请特征} 与{对比特征} 技术差异",
        f"{申请特征} 独特之处"
    ]

    # 使用多个引擎交叉验证
    results_tavily = search_manager.search(citation_queries, [SearchEngineType.TAVILY])
    results_bocha = search_manager.search(difference_queries, [SearchEngineType.BOCHA])

    return merge_results(results_tavily, results_bocha)
```

**输出要求**:
```markdown
【技术理解调研】
对比文件技术分析:
├─ D1 (公开号: CNXXXXXXX)
│  ├─ 技术原理: [深入解释D1的工作机制]
│  ├─ 实现方式: [D1的具体技术手段]
│  ├─ 技术局限: [D1存在的问题或不足]
│  └─ 与本申请区别: [详细技术差异对比]

技术差异证据:
├─ 本申请特征: [本申请的技术特征]
├─ 技术优势: [该特征带来的技术效果]
├─ 实现难度: [该特征的技术难点]
└─ 非显而易见性: [为何该特征不是显而易见的]

反驳依据:
├─ 技术资料支持: [外部技术资料来源]
```

#### 场景3: 专利撰写前的技术调研

**触发条件**: 当用户需要撰写新专利申请时

**搜索目标**:
- 了解技术领域的最新发展
- 查找相关的现有技术用于背景技术描述
- 搜索技术术语的标准表述
- 查找可能影响新颖性的相关专利

**搜索策略**:
```python
def patent_drafting_search(invention_info):
    """
    专利撰写前的技术调研
    """
    queries = {
        'latest_development': f"{invention_info['技术领域']} 最新技术进展",
        'background': f"{invention_info['技术领域']} 背景技术",
        'terminology': f"{invention_info['核心术语']} 专业定义",
        'related_patents': f"{invention_info['技术特征']} 专利"
    }

    # 综合使用多个搜索引擎
    results = {}
    for key, query in queries.items():
        results[key] = search_manager.search(
            query=query,
            engines=[
                SearchEngineType.TAVILY,      # 综合搜索
                SearchEngineType.BOCHA,       # 中文资料
                SearchEngineType.GOOGLE_PATENTS  # 专利检索
            ],
            max_results=5
        )

    return results
```

**输出要求**:

```markdown
├─ 主流方案: [当前主流的技术实现方式]
├─ 技术瓶颈: [现有技术普遍存在的问题]

背景技术参考:
├─ 技术原理: [该领域的基础技术原理]
```

#### 场景4: 技术概念理解和澄清

**触发条件**: 当遇到不懂的技术术语或概念时

**搜索目标**:
- 理解技术术语的准确定义
- 了解技术原理和工作机制
- 查找技术的应用场景
- 确认技术的技术领域分类

**搜索策略**:
```python
def concept_understanding_search(technical_term):
    """
    技术概念理解搜索
    """
    queries = [
        f"{technical_term} 是什么",
        f"{technical_term} 工作原理",
        f"{technical_term} 应用",
        f"{technical_term} 分类"

    # 使用Metaso进行深度AI搜索
    results = search_manager.search(

        engines=[SearchEngineType.METASO],
        max_results=5,
        search_depth="deep"  # 深度搜索模式

    return extract_concept_definition(results)
```

## 【搜索策略标准】

### 标准搜索流程

```
┌─────────────────────────────────────────────────────────────┐
│              步骤1: 确定搜索目标                              │
│  - 明确需要了解什么技术内容                                   │
│  - 确定搜索的关键词和查询词                                   │
│  - 选择合适的搜索引擎                                         │
├─────────────────────────────────────────────────────────────┤
│              步骤2: 执行搜索                                  │
│  - 优先使用Tavily进行综合搜索                                 │
│  - 中文资料使用Bocha补充                                      │
│  - 必要时使用Metaso进行深度搜索                               │
│  - 专利专门信息使用Google Patents                            │
├─────────────────────────────────────────────────────────────┤
│              步骤3: 结果筛选和验证                            │
│  - 相关性过滤: 只保留与问题高度相关的结果                     │
│  - 权威性验证: 优先选择权威来源                               │
│  - 时效性检查: 优先选择最新资料                               │
│  - 多源交叉验证: 对比不同搜索源的结果                         │
├─────────────────────────────────────────────────────────────┤
│              步骤4: 信息整合和输出                            │
│  - 提取关键信息点                                             │
│  - 整合成结构化内容                                           │
│  - 标注数据来源和搜索时间                                     │
│  - 指出信息的局限性                                           │
└─────────────────────────────────────────────────────────────┘
```

### 搜索引擎选择策略

| 业务场景 | 首选引擎 | 辅助引擎 | 备用引擎 |
|---------|---------|---------|---------|
| 背景技术调研 | Tavily | Bocha | Metaso |
| 审查意见答复 | Tavily + Bocha | Metaso | Baidu |
| 专利撰写调研 | Tavily + Google Patents | Bocha | Bing |
| 技术概念理解 | Metaso | Tavily | Bocha |
| 国际专利检索 | Google Patents | Tavily | - |

### 搜索质量标准

**相关性标准**:
- 搜索结果与问题的相关性 > 80%
- 提取的信息点直接回答用户问题
- 避免引入无关信息

**权威性标准**:
- 优先引用权威来源（学术机构、官方文档、行业标准）
- 警惕商业广告或非权威信息
- 标注信息来源的可信度等级

**时效性标准**:
- 技术发展信息优先使用近1-3年资料
- 基础概念可引用经典教材或权威定义
- 标注信息发布时间

**完整性标准**:
- 充分搜索，避免遗漏重要信息
- 多个搜索源交叉验证
- 明确指出信息的局限性

### 协同场景1: 向量库 + 搜索引擎

**场景**: 创造性分析

**协同策略**:

4. **综合推理**: 结合案例经验 + 技术理解 + 法条规定

## 【数据使用约束】

### 1. 搜索触发约束
- **只在需要时搜索**: 不是所有问题都需要搜索引擎
- **先内后外**: 优先使用平台内部数据（向量库、图谱、数据库）
- **明确搜索目标**: 搜索前明确需要了解什么

### 2. 信息质量约束
- **多源验证**: 关键信息必须多源交叉验证
- **权威优先**: 优先选择权威来源
- **时效考虑**: 技术信息注意时效性
- **标注来源**: 必须标注信息来源

### 3. 使用范围约束
- **技术理解**: 搜索引擎主要用于技术知识获取
- **不替代法律检索**: 法律问题仍以向量库和图谱为主
- **辅助判断**: 搜索结果仅供参考，需结合其他数据源

### 4. 成本控制约束
- **API使用**: 注意API调用次数和成本
- **缓存复用**: 相同查询结果应缓存复用
- **精准搜索**: 优化查询词，减少无效搜索

## 【数据质量认知】

### 搜索引擎优势
- **覆盖广泛**: 可获取互联网上的最新技术信息
- **时效性强**: 可获取最新发布的技术资料
- **来源多样**: 学术、工业、标准等多方面信息

### 搜索引擎局限
- **质量不一**: 信息质量参差不齐，需仔细甄别
- **准确性待验证**: 可能包含错误或过时信息
- **权威性不足**: 不如官方文档和学术资料权威
- **API限制**: 受调用次数和速率限制

### 使用原则
- **辅助而非主导**: 搜索引擎是辅助工具，不替代权威数据源
- **验证必做**: 搜索结果必须验证才能使用
- **标注必做**: 必须明确标注信息来源
- **诚实面对**: 遇到不确定要明确说明
