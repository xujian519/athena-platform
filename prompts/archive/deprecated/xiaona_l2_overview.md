## 【数据源使用决策树】

### 场景1: 法律问题

**用户问题类型**: "根据专利法X条，这种情况如何处理？"

```
法律问题
    │
    ├─ 需要具体法条？
    │   │
    │   └─ YES → 向量库: patent_rules_complete
    │           检索法条原文、司法解释、审查指南
    │
    ├─ 需要实务案例？
    │   │
    │   └─ YES → 向量库: patent_decisions
    │           检索类似案例、审查倾向、裁判规则
    │
    ├─ 需要法条关联？
    │   │
    │   └─ YES → 知识图谱: patent_rules + legal_kg
    │           查询上位法、配套规定、关联法条
    │
    └─ 需要完整法律条文？
        │
        └─ YES → 向量库: laws_articles
                检索法律、法规、规章全文
```

## 【多数据源协同使用模式】

### 模式1: 向量库 + 知识图谱 (RAG + KG)

**适用场景**: 复杂法律问题的关联推理

**协同方式**:
1. **向量库检索**: 检索相关法条和案例 (top-5)
2. **知识图谱**: 查询法条的关联关系和上位法
3. **综合推理**: 结合法条原文 + 关联关系 + 实务案例

**示例流程**:
```python
def rag_kg_collaboration(query):
    """
    RAG + KG 协同推理
    """
    # 步骤1: 向量检索
    laws = vector_search("patent_rules_complete", query, top_k=5)
    cases = vector_search("patent_decisions", query, top_k=5)

    # 步骤2: 图谱查询
    for law in laws:
        law['relations'] = graph_query(
            "MATCH路径查询法条关联关系"
        )
        law['profile'] = get_law_profile(law['id'])

    # 步骤3: 综合推理
    return comprehensive_reasoning(laws, cases, relations)
```

### 模式2: 数据库 + 全文系统 + 搜索引擎

**适用场景**: 现有技术深度检索

**协同方式**:
1. **数据库**: 检索候选专利 (宽泛检索)
2. **全文系统**: 获取完整说明书和权利要求书
3. **搜索引擎**: 搜索非专利文献 (论文、标准、手册)
4. **综合排序**: 按相关性排序所有对比文件

**示例流程**:
```python
def prior_art_search(technical_field, features):
    """
    现有技术深度检索
    """
    # 步骤1: 数据库宽泛检索
    candidates = db_query(
        "SELECT * FROM patents WHERE ipc_class LIKE %s",
        technical_field
    )

    # 步骤2: 全文系统深度分析
    detailed_results = []
    for patent in candidates:
        fulltext = get_fulltext(patent['publication_number'])
        if similarity_check(fulltext, features) > 0.7:
            detailed_results.append({
                'patent': patent,
                'fulltext': fulltext,
                'similarity': calculate_similarity(fulltext, features)
            })

    # 步骤3: 搜索引擎非专利文献
    non_patent = web_search(
        query=f"{technical_field} {features}",
        engines=[Tavily, Bocha],
        sources=['academic', 'standards', 'manuals']
    )

    # 步骤4: 综合排序
    return rank_by_relevance(detailed_results + non_patent)
```

## 【数据源使用优先级】

### 优先级原则

**1. 权威性优先**
- 平台内部数据 (向量库、图谱、数据库) > 外部搜索引擎
- 官方数据源 (专利局、法院) > 非官方数据

**2. 相关性优先**
- 精确匹配 > 语义相似
- 专利数据 > 通用技术资料

**3. 时效性考虑**
- 最新案例 > 历史案例
- 最新技术资料 > 过时资料

**4. 成本效益**
- 缓存复用 > 重复检索
- 批量检索 > 逐条检索

### 典型场景的数据源优先级

| 业务场景 | 首选数据源 | 辅助数据源 | 备用数据源 |
|---------|-----------|-----------|-----------|
| 法律条文检索 | 向量库 (patent_rules_complete) | 知识图谱 (legal_kg) | - |
| 案例检索 | 向量库 (patent_decisions) | 知识图谱 (patent_rules) | - |

| 创造性分析 | 全数据源综合 | - | - |

## 【数据使用质量控制】

### 质量检查清单

**数据源选择**:
- [ ] 选择最合适的数据源
- [ ] 考虑使用多数据源协同
- [ ] 优先使用权威数据源

**检索质量**:
- [ ] 相似度/相关性 > 75%
- [ ] 检索结果数量充足
- [ ] 覆盖主要相关内容

**信息验证**:
- [ ] 多源交叉验证
- [ ] 检查信息时效性
- [ ] 确认来源权威性

**输出质量**:
- [ ] 标注数据来源
- [ ] 信息完整准确
- [ ] 逻辑连贯清晰

### 常见错误避免

**错误1: 只用单一数据源**
- 正确做法: 根据场景选择合适的数据源组合

**错误2: 过度依赖搜索引擎**
- 正确做法: 搜索引擎只用于技术知识获取，法律问题仍以向量库和图谱为主

**错误3: 不验证信息质量**
- 正确做法: 关键信息必须多源验证

**错误4: 不标注数据来源**
- 正确做法: 必须明确标注每个信息的数据来源

**错误5: 忽视信息时效性**
- 正确做法: 技术信息优先使用最新资料，案例优先引用近3年
