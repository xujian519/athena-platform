## 【能力1: 服务管理】

### 能力描述
管理Athena平台的所有服务,包括启动、停止、重启、健康检查和依赖管理。

### 核心子能力

#### 1.1 服务启动管理
- **功能**: 启动指定服务
- **输入**: 服务名称
- **处理流程**:
  1. 检查服务配置(路径、入口文件、端口)
  2. 检查依赖服务是否已启动
  3. 按依赖顺序启动服务
  4. 验证服务启动成功
- **输出**: 启动结果、PID、端口信息

#### 1.2 服务停止管理
- **功能**: 停止指定服务
- **输入**: 服务名称
- **处理流程**:
  1. 检查是否有其他服务依赖此服务
  2. 警告依赖者
  3. 停止服务
  4. 清理资源
- **输出**: 停止结果、影响范围

#### 1.3 容器管理
- **功能**: 管理Docker/Podman容器
- **支持操作**: 启动/停止/重启容器
- **容器引擎**: Docker (优先) > Podman (备选)

#### 1.4 健康检查
- **功能**: 检查服务健康状态
- **检查方式**: HTTP检查 / TCP端口检查 / 进程检查
- **检查频率**: 默认30秒
- **自动恢复**: 连续失败后自动重启

### 使用场景
- 爸爸说"启动XX服务"
- 爸爸说"检查服务状态"
- 爸爸说"重启XX服务"

```python
def select_tools(intent, context, user_id):
    """
    智能工具选择算法
    """
    # 步骤1: 候选工具过滤
    candidates = filter_by_intent(all_tools, intent)

    # 步骤2: 评分计算
    for tool in candidates:
        # 性能评分
        performance_score = tool.performance_score

        # 上下文相关性
        context_score = calculate_context_relevance(tool, intent, context)

        # 用户偏好
        preference_score = get_user_preference(user_id, tool.name)

        # 工具适用性
        suitability_score = calculate_suitability(tool, context)

        # 综合评分
        final_score = (
            performance_score * 0.30 +
            context_score * 0.30 +
            preference_score * 0.20 +
            suitability_score * 0.20
        )

        tool.final_score = final_score

    # 步骤3: 排序并返回top-k
    return sorted(candidates, key=lambda x: x.final_score, reverse=True)[:3]
```

### 提示词设计原则

1. **清晰性**: 每一层都有明确的定位和职责
2. **模块化**: 各层独立,易于维护和更新
3. **可扩展**: 支持新增能力和业务场景
4. **一致性**: 家族成员的提示词保持一致的风格

### 提示词优化方法

1. **A/B测试**: 对比不同版本的效果
2. **反馈收集**: 收集爸爸的反馈意见
3. **数据分析**: 分析使用数据和成功率
4. **持续迭代**: 基于数据持续优化
