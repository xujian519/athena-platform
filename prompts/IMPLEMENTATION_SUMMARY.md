### 详细更新内容

#### 1. 中国专利编号体系修正

**问题**: 原提示词中专利号和申请号概念混淆

**修正**: 根据中国国家知识产权局官方规定，明确区分：

| 编号类型 | 产生阶段 | 格式示例 | 用途 |
|---------|---------|---------|------|
| **申请号** | 提交申请时 | CN202310123456.7 | 专利唯一标识 |
| **公开号** | 发明申请公开时 | CN112345678A | 获取全文 |
| **公告号** | 授权公告时 | CN112345678B | 获取全文 |

**关键要点**: 获取专利全文时，需要使用公开号/公告号，而非申请号

#### 2. 专利全文处理系统文档

**新增内容**:
- 系统架构说明（输入层、处理层、存储层、服务层）
- 业务应用说明（现有技术检索、技术特征对比、相似专利发现）
- API使用示例
- 与主数据库的协同方式

**核心作用**: 在专利业务中，专利全文处理系统主要用于获取现有技术（对比文件）

#### 3. 搜索引擎使用场景定义

**四大核心场景**:
1. **创造性分析前的背景技术调研**: 搜索技术领域基础知识、发展历程、常用技术手段
2. **答复审查意见前的技术理解**: 深入理解对比文件、查找反驳依据、搜索支持修改方向的技术资料
3. **专利撰写前的技术调研**: 了解最新发展、查找现有技术、确认技术术语标准表述
4. **技术概念理解和澄清**: 理解技术术语、了解技术原理、确认技术领域分类

**搜索策略**: 优先使用Tavily进行综合搜索，中文资料使用Bocha补充，深度理解使用Metaso，专利专门信息使用Google Patents

#### 4. 多数据源协同模式

**RAG + KG 协同**:
- 向量库检索法条和案例
- 知识图谱查询法条关联关系
- 综合推理给出结论

**数据库 + 全文系统 + 搜索引擎**:
- 数据库检索候选专利
- 全文系统获取完整说明书
- 搜索引擎搜索非专利文献
- 综合排序所有对比文件

**全数据源综合协同**:
- 适用于复杂专利业务
- 结合所有数据源进行综合分析

## 📝 L3能力层新增 - 专利形式审查能力 (2025-12-26)

### 新增能力概述

创建 `cap07_formal_exam.md` - 专利形式审查能力

### 能力特点

**核心定位**: 对专利无效条款的深度理解，特别是形式相关的无效理由

**能力范围**:
1. **发明专利申请初步审查**: 申请文件形式要求检查、明显缺陷识别与处理
2. **实用新型专利申请初步审查**: 申请文件完整性检查、形式要求合规性审查
3. **外观设计专利申请初步审查**: 图片/照片形式检查、简要说明形式审查

**核心价值**:
- 预防性审查: 在申请前识别可能的形式缺陷，避免被驳回
- 申请文件质量保障: 确保申请文件符合法定形式要求
- 审查意见应对: 理解审查员的审查逻辑和驳回依据
- 无效风险预判: 识别可能导致专利无效的形式问题

### 法律基础框架

**核心法条依据**:

1. **诚实信用原则（新增重要条款）**
   - 《专利法实施细则》第十一条
   - 适用场景: 初审驳回、实审驳回、无效宣告理由、依职权宣告无效
   - 具体情形: 虚假提交、编造发明创造、隐瞒重要事实、虚假发明人

2. **申请文件基本要求**
   - 《专利法》第二十六条（各款要求）

3. **不予受理的情形**
   - 《专利法实施细则》第三十九条

4. **初步审查驳回情形**
   - 可补正的缺陷 → 补正通知书
   - 不可克服的缺陷 → 驳回决定

### 常见形式缺陷清单

**A. 申请文件缺陷**:
- 请求书缺陷（发明创造名称、发明人信息、申请人信息）
- 说明书缺陷（必要部分缺失、附图提及问题）
- 权利要求书缺陷（未以说明书为依据、非中文、绘图成分）
- 摘要缺陷（字数超限、商业用语、附图标记）

**B. 明显违反规定的缺陷**:
- 违反专利法第五条（违反法律、社会公德、妨害公共利益）
- 违反专利法第二十五条（不授予专利权的客体）
- 违反诚实信用原则（编造发明、虚假数据、虚假发明人）

**C. 格式和手续缺陷**:
- 格式缺陷（非中文、排版、页码、附图）
- 手续缺陷（委托手续、优先权手续、宽限期手续）

### 特殊场景处理

**场景1: 编造发明创造**
- 识别技术方案明显不完整或不合理
- 依据: 实施细则第十一条（诚实信用原则）

**场景2: 虚假发明人**
- 识别未对发明创造作出实质性贡献的人员
- 依据: 实施细则第十四条 + 第十一条

**场景3: 明显违反法律规定**
- 判断发明创造主题性质
- 依据: 专利法第五条

**场景4: 申请文件格式严重缺陷**
- 检查文件格式和完整性
- 依据: 实施细则第三十九条
