## 一、研究背景

### 1.1 专利分析的重要性

```
专利分析价值:
├─ 新颖性评估: 避免重复研发
├─ 侵权风险: 识别潜在侵权
├─ 技术趋势: 把握行业发展方向
├─ 竞争情报: 了解竞争对手动态
└─ 价值评估: 评估专利资产价值

挑战:
┌─────────────────────────────────────────┐
│  专利文档数量庞大 (>1.3亿件)             │
│  跨语言障碍 (多国专利)                   │
│  专业性强 (技术+法律术语)                │
│  结构复杂 (权利要求+说明书+附图)          │
│  更新频繁 (每年新增数百万件)              │
└─────────────────────────────────────────┘
```

### 1.2 从NLP到多模态AI

```
专利分析技术演进:
├─ 第一代: 规则+统计方法
│   ├─ 关键词匹配
│   ├─ 传统分类器 (SVM, Random Forest)
│   └─ 主题模型 (LDA)
│
├─ 第二代: 深度NLP方法
│   ├─ Word Embeddings (Word2Vec, GloVe)
│   ├─ CNN/RNN/LSTM
│   └─ BERT家族 (PatentBERT, BAAI PatentBERT)
│
└─ 第三代: 多模态AI方法 ⭐当前前沿
    ├─ 文本+图像联合建模
    ├─ 图神经网络 (GNN)
    └─ 生成式大模型 (LLMs, Multimodal LLMs)
```

## 二、专利分析任务分类

### 2.1 五大类任务

```
专利分析任务分类体系:

1. 专利检索 (Patent Retrieval)
   ├─ 关键词检索
   ├─ 语义检索
   ├─ 跨语言检索
   └─ 示例检索 (ByExample)

2. 专利分类 (Patent Classification)
   ├─ IPC分类预测
   ├─ CPC分类预测
   ├─ 技术领域分类
   └─ 多标签分类

3. 专利翻译 (Patent Translation)
   ├─ 中英互译
   ├─ 多语言翻译
   └─ 领域自适应翻译

4. 专利生成 (Patent Generation)
   ├─ 摘要生成
   ├─ 权利要求生成
   ├─ 说明书生成
   └─ 全文草稿生成

5. 其他任务
   ├─ 专利价值评估
   ├─ 专利相似度计算
   ├─ 技术趋势分析
   └─ 侵权检测
```

### 2.2 任务特点

| 任务类型 | 输入 | 输出 | 主要挑战 |
|---------|------|------|----------|
| 检索 | 查询/示例 | 专利列表 | 语义理解、排序 |
| 分类 | 专利文本 | 分类标签 | 多标签、层级分类 |
| 翻译 | 源语言文本 | 目标语言文本 | 术语一致性、长文本 |
| 生成 | 技术描述 | 专利文本 | 法律合规性、技术准确性 |

## 四、数据集与评估

### 4.1 主要数据集

| 数据集 | 规模 | 语言 | 任务 |
|--------|------|------|------|
| CLEF-IP | ~100万 | 英/多 | 检索 |
| TREC-CHEM | ~100万 | 英 | 检索 |
| PatentsView | 700万+ | 英 | 多任务 |
| CNIPA | 4000万+ | 中 | 多任务 |
| Google Patents | 1.3亿+ | 多 | 多任务 |

### 4.2 评估指标

```
专利检索评估:
├─ Precision@K: 前K个结果准确率
├─ Recall@K: 前K个结果召回率
├─ MAP: 平均精度均值
├─ nDCG: 归一化折损累积增益
└─ ERR: 期望倒数排名

专利分类评估:
├─ Accuracy: 准确率
├─ F1-score: F1分数
├─ Precision/Recall: 精确率/召回率
└─ Hierarchical Metrics: 层级分类指标

专利生成评估:
├─ BLEU: 机器翻译常用指标
├─ ROUGE: 摘要生成常用指标
├─ METEOR: 语义相似度
├─ BERTScore: 基于BERT的语义相似度
└─ 人工评估: 法律合规性、技术准确性
```

