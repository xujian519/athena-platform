### 📊 论文概览

**研究类型**: 方法论论文 + 专利专用大语言模型

**核心贡献**:
1. 提出Patent-GPT - 专利领域专用的短语嵌入模型
2. 整合专利引文网络与语义相似度
3. 在专利检索任务上超越现有SOTA方法
4. 验证了GPT架构在专利短语表示学习中的有效性

### 🎯 核心内容

#### 3.1 专利短语嵌入的挑战

**传统问题**:
```
专利短语特点:
├─ 长度差异大: 从几个词到完整句子
├─ 技术术语密集: 包含大量专业词汇
├─ 法律语言严谨: 需要精确的法律表述
├─ 语义细微差别: 相似词语在不同上下文有不同含义
└─ 上下文依赖强: 需要考虑专利文档的整体语境
```

#### 3.2 Patent-GPT架构

**模型设计**:
```
Patent-GPT架构:

┌─────────────────────────────────────────────────────────┐
│                    Patent-GPT 模型                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  [输入层] Input Layer                                    │
│    ├─ 专利短语: "一种基于深度学习的图像识别方法"        │
│    ├─ 专利权利要求: "1. 一种...方法,其特征在于..."       │
│    └─ 专利摘要: 完整专利摘要文本                         │
│                                                         │
│  [编码层] Encoding Layer (基于GPT架构)                   │
│    ├─ Token嵌入层                                       │
│    ├─ 位置编码层                                        │
│    ├─ 多头自注意力机制 (Multi-head Self-Attention)      │
│    ├─ 前馈神经网络层                                     │
│    └─ 层归一化 (Layer Normalization)                     │
│                                                         │
│  [输出层] Output Layer                                   │
│    ├─ 短语嵌入: 固定维度向量表示                        │
│    ├─ 语义相似度: 计算短语间相似度                       │
│    └─ 引文预测: 预测专利间引用关系                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 3.3 多任务学习框架

**训练任务**:
```
三大任务联合训练:

1. 短语相似度学习
   ├─ 目标: 相似短语产生相近嵌入
   ├─ 损失: 对比损失 (Contrastive Loss)
   └─ 数据: 专利短语对 (相似/不相似)

2. 引文关系预测
   ├─ 目标: 预测专利间是否存在引用关系
   ├─ 损失: 二元交叉熵损失
   └─ 数据: 专利引文网络

3. 专利分类任务
   ├─ 目标: 将专利分配到正确的IPC分类
   ├─ 损失: 交叉熵损失
   └─ 数据: 带有IPC标签的专利数据
```

#### 3.4 融合专利引文网络

**引文增强机制**:
```
引文网络融合策略:

图神经网络层
    ↓
专利引文图
    ↓
与语义嵌入融合
    ↓
增强的短语表示

优势:
├─ 引入结构化知识
├─ 捕获专利间技术传承关系
├─ 改善检索质量
└─ 提升分类准确性
```

### 📈 关键发现与数据

#### 4.1 实验结果

**专利检索性能**:
| 模型 | Recall@10 | Recall@100 | MAP | nDCG@10 |
|------|-----------|------------|-----|---------|
| BM25 | 0.582 | 0.812 | 0.456 | 0.623 |
| Sentence-BERT | 0.671 | 0.876 | 0.534 | 0.701 |
| PatentBERT | 0.723 | 0.901 | 0.598 | 0.756 |
| **Patent-GPT** | **0.789** | **0.934** | **0.645** | **0.812** |

**专利分类性能**:
| 模型 | Accuracy | Micro-F1 | Macro-F1 |
|------|----------|----------|----------|
| BERT-base | 0.712 | 0.698 | 0.534 |
| SciBERT | 0.756 | 0.743 | 0.589 |
| PatentBERT | 0.798 | 0.785 | 0.623 |
| **Patent-GPT** | **0.834** | **0.821** | **0.678** |

#### 4.2 消融实验

**组件贡献分析**:
```
配置召回率分析:
├─ 基础GPT模型: Recall@10 = 0.723
├─ + 引文网络: Recall@10 = 0.756 (+4.5%)
├─ + 多任务学习: Recall@10 = 0.778 (+2.2%)
└─ Patent-GPT (完整): Recall@10 = 0.789 (+1.1%)

结论: 引文网络贡献最大,多任务学习次之
```

#### 4.3 案例分析

**检索质量提升**:
```
查询: "深度学习图像识别"
相关专利排序:

传统方法 (BM25):
1. 基于CNN的图像处理方法
2. 图像特征提取技术
3. 深度神经网络应用
   ↓ 关键专利排在第7位

Patent-GPT:
1. 深度学习图像识别方法 ⭐
2. 卷积神经网络图像处理
3. 多层深度学习模型
   ↓ 关键专利排在第1位
```
