# 专利混合检索系统 Web界面使用说明

## 🎯 系统简介

专利混合检索系统是一个现代化的专利检索平台，整合了三种检索策略：
- **BM25全文搜索**: 基于关键词匹配的传统检索
- **向量语义检索**: 基于embedding的语义相似度检索  
- **知识图谱增强**: 利用专利关联关系的图检索

## 🚀 快速启动

### 方法一：一键启动（推荐）

```bash
cd patent-retrieval-webui
./start.sh
```

这将同时启动前端和后端服务。

### 方法二：分别启动

**启动后端：**
```bash
cd patent-retrieval-webui/backend
./start_server.sh
```

**启动前端（新开一个终端）：**
```bash
cd patent-retrieval-webui
npm run dev
```

## 📍 访问地址

启动成功后，访问以下地址：

- **前端界面**: http://localhost:3000
- **后端API**: http://localhost:8000
- **API文档**: http://localhost:8000/docs

## 📖 功能使用指南

### 1. 专利检索

进入主页，在检索框中输入查询内容：

```
示例查询：
- "深度学习图像识别"
- "区块链数据存储"
- "自然语言处理技术"
```

**高级筛选：**
- **返回结果数量**: 选择10/20/50/100条
- **IPC分类代码**: 输入分类号，如"G06F, H04L"
- **申请人**: 输入申请人或公司名称
- **发布日期范围**: 选择开始和结束日期

点击"开始检索"按钮，系统会并行执行三种检索策略，并融合返回结果。

### 2. 查看检索结果

检索结果页面显示：
- 专利标题和ID
- 综合评分（分数越高越相关）
- 数据来源（FT/BM25、VEC/向量、KG/知识图谱）
- 专利摘要
- 匹配证据（高亮显示匹配的文本）
- 评分明细（各检索策略的贡献度）

点击任意专利卡片可以查看详细信息。

### 3. 搜索历史

点击顶部导航的"历史"按钮，可以：
- 查看所有搜索记录
- 点击历史记录重新执行搜索
- 删除单条历史记录
- 清空所有历史记录

历史记录包含：
- 搜索时间
- 查询内容
- 结果数量
- 前三个结果预览

### 4. 权重配置

点击"配置"按钮进入配置页面，可以：

**调整检索权重：**
- 拖动滑块调整三种检索策略的权重
- 权重总和必须为100%（1.0）
- 权重调整后实时生效

**查看系统状态：**
- PostgreSQL连接状态
- Qdrant连接状态
- Neo4j连接状态
- 专利总数

**默认权重配置：**
- BM25全文搜索: 40%
- 向量语义检索: 50%
- 知识图谱增强: 10%

### 5. 实时监控

点击"监控"按钮进入监控页面，可以查看：

**性能指标：**
- 缓存命中率
- 平均响应时间
- 活跃连接数
- 错误率
- 每分钟查询数

**可视化图表：**
- 查询频率趋势图
- 性能指标趋势图（响应时间、缓存命中率）

监控数据每5秒自动更新。

## 💡 使用技巧

### 优化检索效果

1. **使用具体的技术关键词**
   - ❌ "好的东西"
   - ✅ "深度学习卷积神经网络"

2. **合理使用IPC分类筛选**
   - G06F: 电子数据处理
   - H04L: 数字信息传输
   - G06N: 人工智能模型

3. **根据场景调整权重**
   - 专利侵权分析 → 提高向量检索权重（语义理解）
   - 技术文献检索 → 提高全文搜索权重（精确匹配）
   - 技术趋势分析 → 提高知识图谱权重（关联关系）

### 高级功能

1. **批量操作**: 搜索历史支持回放和删除
2. **实时监控**: 监控页面自动刷新，无需手动刷新
3. **响应式设计**: 支持手机、平板、桌面等设备

## 🐛 常见问题

### Q: 检索返回空结果？

A: 检查以下几点：
- 数据库是否已初始化
- 检索关键词是否合理
- 筛选条件是否过于严格
- 查看后端日志排查错误

### Q: 前端无法连接后端？

A: 检查：
- 后端服务是否正常运行
- 浏览器控制台是否有错误信息
- 防火墙是否阻止了8000端口

### Q: 权重配置无法保存？

A: 确保权重总和为1.0，每个权重在0-1之间。

### Q: 监控数据不更新？

A: 监控数据每5秒自动刷新，如果长时间不更新，刷新浏览器页面。
