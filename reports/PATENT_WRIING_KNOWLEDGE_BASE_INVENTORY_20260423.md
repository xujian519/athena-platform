3. 充分公开的判断标准：所属领域技术人员不需创造性劳动即可实现
4. 未充分公开的三大情形：目的不清楚、技术方案达不到目的、缺少必要实施条件

### 文件3: 撰写-审查要点-权利要求书撰写-示例与常见问题

**来源**: 黄敏《专利申请文件的撰写与审查要点》第二部分第一章

**核心内容**:
- 权利要求必须清楚、简要地表述请求保护的范围
- "清楚"的两层含义：类型清楚（产品/方法）、用词清楚（技术术语、不含糊词汇）
- "简要"的含义：除记载技术特征外，不得对原因或理由作不必要描述
- 两部分撰写形式：前序部分（发明名称+共有必要特征） + 特征部分（"其特征是"+区别特征）
- 划界的概念与创造性的关系
- 从属权利要求规范：引用部分+限定部分、只能用"或"不能用"和"

**提取的关键知识点**:
1. 产品权利要求与方法权利要求的保护差异（产品绝对、方法相对）
2. 禁止用语：例如、最好是、尤其是、必要时、约、接近、厚薄、强弱、高温高压
3. 两部分撰写形式不适用情形：开拓性发明、等效组合发明、省略改进发明、复杂系统
4. 多项从属权利要求禁止"多项引多项"

**适用性**: 全领域

### 文件4: 撰写-审查要点-权利要求书撰写-以说明书为依据

**来源**: 黄敏《专利申请文件的撰写与审查要点》第二部分第一章

**核心内容**:

4. 功能性特征允许情形（停止位置检测装置-光电管或应变片替代限位开关）
5. 功能性特征禁止情形（彩色显像管-特定方式实现，无其他替代实施例）

### 文件6: 撰写-审查要点-创造性判断要点-上

**来源**: 黄敏《专利申请文件的撰写与审查要点》第二部分第二章

**核心内容**:
- 创造性判断采用组合对比原则：可将两份或多份对比文件组合
- 创造性的基本定义：发明对所属领域技术人员非显而易见
- 显而易见组合的判断标准：文件性质和内容、技术领域差异、组合数量
- 整体考虑原则：技术方案作为整体，功能关系必须显而易见
- 三步法判断思路：确定最接近现有技术 → 确定区别特征和技术问题 → 判断是否显而易见

**提取的关键知识点**:
1. 组合对比原则 vs 单独对比原则的区别
2. 显而易见组合的3方面考虑（文件关联性、领域相近性、文件数量）
3. 整体考虑原则的重要性（特征功能联系vs简单特征堆砌）
4. 技术启示的概念：现有技术给出应用区别特征的启示
5. 事后诸葛亮误区：避免以已知结果倒推显而易见性

**适用性**: 全领域

### 文件7: 撰写-机械-说明书撰写-发明内容与实施方式

**来源**: 张荣彦《机械领域专利申请文件的撰写与审查》第一章

**核心内容**:
- 发明内容三个组成部分：要解决的技术问题、技术方案、有益效果
- 三者必须相互对应，不能矛盾或不相关
- 有益效果应具体分析，不能只给断言，不得使用广告性语言
- 具体实施方式是充分公开的关键，应详细描述优选方式
- 附图标记规范：阿拉伯数字、一致性、位置规范

**提取的关键知识点**:
1. 技术问题的表述：针对现有技术缺陷、正面简洁、与背景技术对应
2. 技术方案的表述：技术措施的集合，由技术特征体现
3. 有益效果的表述：技术特征直接带来或必然产生，具体分析，禁用广告语言
4. 实施方式的要求：详细具体、结合附图、多个变型（概括性特征）
5. 附图标记规范：同一标记表示同一组成部分、摘要加括号

**适用性**: 机械、化学、电子（生物和软件需额外考虑实验数据）

### 文件8: 撰写-审查要点-权利要求书撰写-必要技术特征

**来源**: 黄敏《专利申请文件的撰写与审查要点》第二部分第一章

**核心内容**:
- 必要技术特征的定义：解决技术问题所不可缺少的技术特征
- 独立权利要求应当记载解决技术问题的全部必要技术特征
- 从技术问题出发反推必要特征的3步法
- 最接近现有技术的选择标准：技术领域相同/相近、技术问题/效果/用途最接近
- 前序部分和特征部分的划界依据：最接近现有技术（发明原型）

**提取的关键知识点**:
1. 必要技术特征的判断标准：缺少该特征就无法解决技术问题
2. 前序部分特征：与现有技术共有的必要技术特征
3. 特征部分特征：区别于现有技术的技术特征
4. 划界不当的后果：特征部分越少，创造性可能性越小
5. 非必要特征的处理：写入从属权利要求，不应写入独立权利要求

**适用性**: 全领域

### 文件9: 撰写-常见错误-水龙头

**来源**: 吴观乐《发明和实用新型专利申请文件撰写案例剖析》第三部分案例六

- 从属权利要求主题名称不一致："水龙头的阀芯位置锁定装置" vs "水龙头"

**提取的关键知识点**:

3. 从属权利要求主题名称必须与引用权利要求一致
4. 技术领域不写区别特征，只写上位技术领域
5. 独立权利要求只写必要特征，附加特征写入从属权利要求

## 三、提取的关键知识点汇总

### 3.1 撰写前准备（5个知识点）

1. **实用性条件判断**: 能够在工业上制造或使用 + 能够产生积极效果
2. **技术方案要求**: 涉及技术领域、技术问题、符合的技术特征
3. **充分公开标准**: 所属领域技术人员不需创造性劳动即可实现
4. **未充分公开情形**: 目的不清楚、技术方案达不到目的、缺少必要实施条件
5. **专利保护对象**: 技术方案（产品/方法），而非构思或物理状态

### 3.2 说明书撰写（12个知识点）

6. **八大部分撰写规范**: 名称≤25字、技术领域具体、背景技术引证最接近现有技术、目的正面简洁、技术方案清晰、效果具体分析、图面说明集中、实施方式详细
7. **技术问题表述**: 针对现有技术缺陷、正面简洁、与背景技术对应
8. **技术方案表述**: 技术措施的集合，由技术特征体现，能够达到目的
9. **有益效果表述**: 技术特征直接带来或必然产生，具体分析，禁用广告语言
10. **三者对应关系**: 技术问题、技术方案、有益效果必须相互对应
11. **实施例数量判断**: 简单发明可省略、宽范围权利要求需多个实施例、数值范围需两端值+中间值
12. **附图标记规范**: 阿拉伯数字、同一标记表示同一组成部分、摘要加括号
13. **概括性特征处理**: 应给出几个实施方式，除非对本领域技术人员显而易见
14. **机械领域特殊要求**: 一般不必给出具体尺寸，除非有特定选择含义
15. **化学领域特殊要求**: 需借助实验数据说明效果，给出实验条件和方法
16. **摘要要求**: ≤300字、一附图、不得商业宣传
17. **常见错误**: 三者不对应、效果笼统、广告语言、描述过于简单、缺少关键实施例

### 3.3 权利要求书撰写（18个知识点）

18. **清楚简要要求**: 清楚（类型清楚、用词清楚）+ 简要（不描述原因理由）
19. **类型清楚**: 产品权利要求 vs 方法权利要求，保护差异
20. **用词清楚**: 禁止土语、自造词语、含糊词汇（厚薄、强弱、高温高压、很宽范围）
21. **禁止用语**: 例如、最好是、尤其是、必要时、约、接近
22. **两部分撰写形式**: 前序部分（发明名称+共有必要特征） + 特征部分（"其特征是"+区别特征）
23. **划界概念**: 以最接近现有技术（发明原型）为依据
24. **划界与创造性关系**: 特征部分特征越少，创造性可能性越小
25. **不适用两部分形式**: 开拓性发明、等效组合发明、省略改进发明、复杂系统
26. **从属权利要求形式**: 引用部分（引用权利要求编号+主题名称） + 限定部分（附加技术特征）
27. **引用规范**: 多项引用只能用"或"不能用"和"，多项从属权利要求禁止"多项引多项"
28. **句号使用**: 每一项权利要求只允许在其结尾处使用句号
29. **以说明书为依据**: 用语对应（绝对性） + 实施例支持（相对性）
30. **用语对应核查**: 权利要求记载内容在说明书中是否有相同或相应描述
31. **实施例支持核查**: 所属领域技术人员能否不付出创造性劳动实现权利要求概括的全部技术方案
32. **功能性限定允许情形**: 所属领域技术人员知道替代手段，一个实施例即可
33. **功能性限定禁止情形**: 特定方式实现，无其他替代实施例，或说明含糊不可知
34. **简单发明实施例**: 背景技术部分已清楚反映实施方式，无需额外实施例
35. **必要技术特征定义**: 解决技术问题所不可缺少的技术特征
36. **前序部分特征**: 与现有技术共有的必要技术特征
37. **特征部分特征**: 区别于现有技术的技术特征

### 3.4 新颖性创造性判断（10个知识点）

38. **新颖性单独对比原则**: 一份对比文件中的一个技术方案
39. **信息类型**: 出版物、公开使用、其它公开方式、抵触申请
40. **时间地域界限**: 申请日以前（不含当天），出版物全球、使用仅国内
41. **上下位概念规则**: 一般不损害具体，具体损害一般，具体不损害具体
42. **惯用手段置换**: 螺钉vs螺栓等惯用手段置换不具新颖性
43. **数值范围判断**: 7种情形（两端值、具体数值、单点、共同端值、无重叠、选择发明、排除性范围）
44. **创造性组合对比原则**: 可将两份或多份对比文件组合
45. **显而易见性判断**: 文件性质和内容、技术领域差异、组合数量
46. **整体考虑原则**: 技术方案作为整体，功能关系必须显而易见
47. **三步法判断**: 最接近现有技术 → 区别特征和技术问题 → 是否显而易见

### 3.5 常见错误（5个知识点）

48. **缺少必要技术特征**: 从技术问题出发反推必要特征
49. **主题名称不一致**: 从属权利要求主题名称必须与引用权利要求一致
50. **技术领域过窄**: 不写区别特征，只写上位技术领域
51. **写入非必要特征**: 独立权利要求只写必要特征，附加特征写入从属权利要求
52. **技术问题与特征对照表**: 打开水龙头清洗手柄 → 阀杆伸出出水通道

### 3.6 案例启示（5个知识点）

53. **核心改进点提炼**: 对比现有技术，找出区别技术特征
54. **多层次保护体系**: 独立权利要求概括核心改进，从属权利要求层层递进
55. **功能性概括技巧**: "连接位置邻近端部"而非限定具体连接方式
56. **背景技术对比充分**: 详细描述现有技术缺陷，突出发明贡献
57. **实施方式完整性**: 两种实施方式支持权利要求中的概括性描述

### 4.3 知识卡片分类体系

建议建立4级分类体系：

**一级分类**（大类）:
- 撰写规则
- 常见错误
- 判断标准
- 案例参考

**二级分类**（子类）:
- 撰写规则: 说明书、权利要求书、附图、摘要
- 常见错误: 语言表达、逻辑关系、格式规范、内容缺陷
- 判断标准: 新颖性、创造性、实用性、充分公开
- 案例参考: 机械、化学、电子、软件、生物

**三级分类**（主题）:
- 说明书: 发明内容、实施方式、技术领域、背景技术
- 权利要求书: 独立权利要求、从属权利要求、必要技术特征、功能性限定
- 语言表达: 模糊词汇、主观词汇、非限定词汇
- 逻辑关系: 术语不一致、引用错误、因果关系错误

**四级分类**（标签）:
- 严重程度: 致命缺陷、严重缺陷、一般缺陷、轻微缺陷
- 出现频率: 高频（>30%）、中频（10-30%）、低频（<10%）
- 检测难度: 易发现、需仔细检查、需专业判断
- 可修复性: 可修复、部分修复、不可修复

## 六、风险提示

### 6.1 文件缺失风险

**缺失文件影响**:
- 语言表达规范缺失 → 无法自动检测模糊词汇（"约"、"大约"）
- 逻辑关系规范缺失 → 无法自动检测引用错误和术语不一致
- 格式规范缺失 → 无法自动检查权利要求编号格式
- 化学/软件/生物案例缺失 → 知识卡片覆盖领域不全

**缓解措施**:
1. 从现有案例中提取错误模式，补充常见错误规范
2. 从审查指南和审查案例中提取规则，补充格式规范
3. 从化学撰写文件中提取典型案例，补充案例库

### 6.2 质量控制风险

**质量问题**:
- 知识点提取不完整或不准确
- 分类体系不合理导致查询困难
- 案例与规则匹配错误

**质量控制措施**:
1. 建立双人审核机制（AI提取 + 人工审核）
2. 建立测试用例库验证知识卡片准确性
3. 定期回顾和更新知识卡片内容

### 6.3 整合风险

**整合挑战**:
- 知识卡片格式与小娜代理提示词系统不兼容
- 知识卡片数量过多导致提示词超长
- 实时检查功能影响撰写效率

**整合策略**:
1. 设计分模块加载机制，按需加载相关知识卡片
2. 建立优先级排序，优先加载高优先级卡片
3. 开发异步检查机制，不影响用户实时撰写体验
