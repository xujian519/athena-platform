### 2.2 语义增强

```python
class SemanticEnhancer:
    def enhance_vector(self, text, metadata):
        # 1. 基础embedding
        base_vector = self.bge_model.encode(text)
        # 2. 上下文扩展
        context = self._expand_context(text, metadata)
        # 3. 关键词加权
        keywords = self._extract_legal_keywords(text)
        # 4. 实体识别增强
        entities = self._extract_entities(text)
        # 5. 多向量融合
        return self._fuse_vectors([
            base_vector, context, keywords, entities
        ])
```

## 🔗 四、知识图谱引入评估

### 4.1 向量检索的局限性

| 局限性 | 影响 | 严重程度 |
|--------|------|----------|
| 关系缺失 | 无法识别引用关系 | 🔴 高 |
| 推理能力弱 | 无法多跳推理 | 🔴 高 |
| 实体歧义 | 同实体不同表述 | 🟡 中 |
| 时序缺失 | 无法追踪演变 | 🟡 中 |

### 4.2 知识图谱增强价值

**核心实体类型**:
```cypher
(:Law)-[:HAS_ARTICLE]->(:Article)
(:Case)-[:CITES]->(:Article)
(:Patent)-[:INVALIDATED_BY]->(:Case)
(:Article)-[:REFERENCED_IN]->(:Case)
```

**融合检索策略**:
1. 纯向量检索: 语义相似度
2. 纯图谱检索: 关系推理
3. 向量引导图谱: 种子扩展
4. 图谱剪枝向量: 范围限定
5. 融合双路: 最优效果 ⭐

### 4.3 应用场景

**场景1: 法条关联推荐**
- 展示条文间的关系
- 追踪引用链条
- 推荐相关案例

**场景2: 案例先例链追踪**
- 查找先例案例
- 追踪后续引用
- 分析裁判规则演变

**场景3: 多跳推理**
- 法律推理路径
- 因果关系分析
- 判决逻辑解释

### 4.4 引入建议

**✅ 强烈建议引入知识图谱**

**理由**:
1. ✅ 数据天然具有图结构
