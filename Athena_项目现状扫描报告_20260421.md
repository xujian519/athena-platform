## 🟠 架构不合理之处（高优先级）

### 1. Gateway与Core职责边界不清 ⭐⭐⭐⭐⭐

**问题严重性**：★★★★★（架构缺陷，扩展困难）

**问题描述**：
| 模块 | 职责 | 问题 |
|------|------|------|
| `gateway-unified/` | Go语言网关服务 | 包含业务逻辑 |
| `core/` | Python核心逻辑 | 包含路由和通信 |

**具体证据**：
```go
// gateway-unified/internal/middleware/ - Go语言中间件
// 与 core/middleware/ - Python语言中间件重复
```

**优化建议**：
1. **明确职责**：
   - Gateway只负责：网络通信、协议转换、负载均衡
   - Core只负责：业务逻辑、Agent调度、数据处理
2. **建立清晰接口**：
   - Gateway → Core 的REST/gRPC接口
   - 统一的错误处理和日志格式
3. **逐步迁移**：
   - 将Gateway中的业务逻辑迁移到Core
   - 将Core中的网络逻辑迁移到Gateway

---

### 2. Domains与Core业务划分混乱 ⭐⭐⭐⭐

**问题严重性**：★★★★☆（逻辑混乱，错误率高）

**问题描述**：

   - `core/`：通用框架和基础设施
   - `domains/`：业务领域特定实现
2. **迁移计划**：
   - 将 `core/legal_*` 迁移到 `domains/legal-ai/`
   - 将 `core/patent/` 迁移到 `domains/patent-ai/`
3. **建立依赖规则**：
   - Domains可以依赖Core
   - Core不能依赖Domains
