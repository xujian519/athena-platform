## 📊 四大核心基础设施详解

### 1. 法律知识图谱 (NebulaGraph)
**位置**: `/nebula-graph-deploy/`
**功能**: 存储法律实体和关系的图数据库

```yaml
核心实体:
  - legal_document: 法律文档 (3,015个)
  - legal_article: 法律条文
  - legal_category: 法律分类
  - legal_case: 司法案例
  - legal_concept: 法律概念
  - legal_institution: 法律机构

核心关系:
  - contains: 包含关系 (文档->条文)
  - belongs_to: 分类关系
  - references: 引用关系
  - amends: 修改关系
  - interprets: 解释关系
```

### 2. 专业向量库 (Vector Database)
**位置**: `/tools/Laws-1.0.0/db.sqlite3`
**功能**: 存储法律文本的向量表示，支持语义搜索

```yaml
数据规模:
  - 法律文档: 3,080部
  - 向量维度: 768维 (BERT)
  - 支持语言: 中文
  - 相似度算法: 余弦相似度

搜索能力:
  - 语义搜索: 理解查询意图
  - 相似度匹配: 0.8+高精度
  - 混合搜索: 向量+图谱+关键词
```

### 3. 存储系统 (Storage System)
**位置**: 多层级存储架构
**功能**: 提供高效、可靠的数据存储服务

```yaml
存储层级:
  L1: 内存缓存 (Redis) - 毫秒级访问
  L2: 向量数据库 (SQLite+pgvector) - 语义搜索
  L3: 图数据库 (NebulaGraph) - 关系查询
  L4: 文件系统 (本地/云) - 原始文档

性能指标:
  - 缓存命中率: 89.7%
  - 查询响应: <100ms (95%)
  - 存储容量: 支持TB级扩展
```

### 4. 记忆模块 (Memory Module)
**位置**: 分布式记忆系统
**功能**: 为小诺提供长期记忆和上下文管理

```yaml
记忆类型:
  - 对话记忆: 会话历史管理
  - 知识记忆: 学习的知识点
  - 事实记忆: 用户偏好和上下文
  - 推理记忆: 推理链条缓存

记忆管理:
  - 自动归档: 7天未访问自动归档
  - 智能检索: 基于相似度的快速检索
  - 隐私保护: 敏感信息加密存储
```

## 🔄 基础设施协同工作流程

```

    ↓

```
