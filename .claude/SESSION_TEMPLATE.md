# 🚀 Claude Code 会话级快速配置

> **使用方法**: 在每次Claude Code会话开始时复制相关内容

---

## 🎯 会话配置模板

### 模板1: 标准开发会话

```markdown
## 本次会话配置

**项目**: Athena工作平台
**任务**: {请填写具体任务}
**上下文**: {相关文件、模块、功能}

### 分析要求
- 请先阅读 .claude-project-context.md 了解项目背景
- 以资深架构师视角进行分析
- 使用思维链逐步思考复杂问题

### 输出要求
- 使用结构化的Markdown格式
- 代码使用英文编写，附带中文注释
- 包含具体可执行的建议

### 约束条件
- 遵循项目技术栈 (Python + PostgreSQL + Qdrant + Neo4j)
- 遵循PEP 8代码规范
- 考虑Gateway架构原则

### 其他
如有特殊要求，请在此说明：
_例如: 本次会话不考虑向后兼容 / 本次会话优先考虑性能而非可维护性等_
```

### 模板2: 快速问题解答

```markdown
## 快速咨询

**项目**: Athena工作平台
**问题类型**: 快速解答

### 回答要求
- 直接回答，不需要详细分析
- 可以简洁，但要准确
- 如果需要，可以提供相关文档链接

### 背景信息
{必要的背景信息}
```

### 模板3: 深度技术分析

```markdown
## 深度技术分析

**项目**: Athena工作平台
**任务**: {技术分析任务}
**深度**: Super Thinking Mode (9个AI代理协同)

### 分析范围
1. **全息解构** (25%): 360度无死角分析
2. **前瞻性思考** (30%): 未来3-6个月发展预判
3. **跨域融合** (25%): 多学科知识整合
4. **决策支持** (20%): C级别决策建议包

### 输出标准
- 5000-8000字的深度分析报告
- 完整的决策支持包
- 多方案对比和选择建议
- 长期战略规划建议

### 分析框架
请使用以下框架进行分析：

#### 角色分配
- 后端架构师视角: 可扩展性、性能
- 前端工程师视角: 用户体验、接口设计
- DevOps工程师视角: 部署、监控、维护
- 安全专家视角: 安全风险、缓解措施

#### 思维过程
1. **问题理解**: 核心问题是什么？
2. **多维分析**: 技术、业务、风险
3. **方案对比**: 至少2个可行方案
4. **推理过程**: 让我们逐步思考...
5. **结论建议**: 推荐方案及理由

### 质量检查
- [ ] 分析深度足够
- [ ] 考虑了多个角度
- [ ] 提供了具体建议
- [ ] 评估了潜在风险
```

### 模板4: 代码审查

```markdown
## 代码审查会话

**项目**: Athena工作平台
**任务**: 审查以下代码

### 审查要求
- 代码质量: PEP 8规范、可读性、可维护性
- 功能正确性: 是否符合需求
- 性能考虑: 是否有性能问题
- 安全风险: 是否有安全隐患
- 测试覆盖: 是否有足够测试

### 代码内容
{粘贴代码或指定文件路径}

### 审查输出
1. **总体评价**: 代码质量评分 (1-10)
2. **问题清单**: 发现的问题列表
3. **改进建议**: 具体的改进建议
4. **修复代码**: 如需要，提供修复后的代码

### 审查角色
请以资深Python开发者的视角进行审查，严格但不苛刻。
```

### 模板5: Bug诊断与修复

```markdown
## Bug诊断会话

**项目**: Athena工作平台
**任务**: 诊断并修复Bug

### Bug描述
{bug描述}

### 错误信息
```
{粘贴错误信息}
```

### 诊断要求
1. **问题定位**: 哪里出错了？为什么？
2. **根因分析**: 深层原因是什么？
3. **解决方案**: 提供修复代码
4. **测试验证**: 如何验证修复？

### 修复要求
- 提供完整的修复代码（英文代码 + 中文注释）
- 说明修复原理
- 提供测试用例
- 评估修复风险

### 约束条件
- 不能破坏现有功能
- 需要保持代码可读性
- 需要添加测试（如果没有）
```

## 🔧 会话级快速调整

### 调整1: 临时降低分析深度

```markdown
## 临时调整

本次会话：
- 分析深度: 快速模式（不需要详细分析）
- 输出长度: 简洁扼要
- 跳过思维链步骤

其他项目配置保持不变。
```

### 调整2: 临时改变关注点

```markdown
## 临时调整

本次会话：
- 关注重点: 性能优化（暂不考虑可维护性）
- 输出要求: 优先性能数据
- 约束条件: 可以使用高级特性，不考虑兼容性

其他项目配置保持不变。
```

### 调整3: 临时切换技术栈

```markdown
## 临时调整

本次会话：
- 技术栈: 考虑MySQL方案（虽然项目主要使用PostgreSQL）
- 分析角度: 对比MySQL vs PostgreSQL
- 输出要求: 提供对比分析结果

其他项目配置保持不变。
```

## 🚀 快速开始

### 立即使用

1. **选择模板**: 根据您的任务选择合适的模板
2. **复制粘贴**: 复制模板内容到Claude Code
3. **填写信息**: 填写{...}占位符内容
4. **开始对话**: 发送配置，开始会话

### 示例

#### 示例1: 分析Gateway架构
```markdown
## 深度技术分析

项目: Athena工作平台
任务: 分析Gateway架构设计

请参考OpenClaw的Gateway架构，分析Athena的Gateway设计：
1. 架构对比
2. 优势劣势
3. 改进建议
4. 实施路线
```

#### 示例2: 修复Bug
```markdown
## Bug诊断会话

项目: Athena工作平台
任务: 修复PostgreSQL连接问题

Bug描述:
- 错误: "connection refused" when connecting to PostgreSQL
- 场景: 首次启动Docker服务
- 最近更改: 更新了docker-compose.yml

请诊断问题并提供解决方案。
```
