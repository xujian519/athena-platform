# 🎯 专利检索系统实施方案 v2.0
## 基于PostgreSQL+pgvector的三阶段混合检索系统

### 📋 项目概述

基于对您项目搜索实施方案的深度分析，结合当前38,543,908条专利数据的现状，我为您制定了本方案，旨在构建一个避免漏检的高精度专利检索系统。

### 🎯 核心目标

1. **避免漏检**: 通过多路召回策略，确保相关专利不被遗漏
2. **精准排序**: 使用向量检索+关键词检索+LLM重排的三阶段架构
3. **本地化部署**: 所有数据和模型本地处理，满足合规要求
4. **高扩展性**: 支持4000万+数据规模的高效检索

#### 1.2 添加向量字段
```sql
-- 为专利表添加向量字段（768维，适合BERT模型）
ALTER TABLE patents
ADD COLUMN embedding_title VECTOR(768),
ADD COLUMN embedding_abstract VECTOR(768),
ADD COLUMN embedding_claims VECTOR(768),
ADD COLUMN embedding_combined VECTOR(768);

-- 创建向量索引（使用HNSW算法）
CREATE INDEX CONCURRENTLY idx_patents_embedding_combined_hnsw
ON patents USING hnsw (embedding_combined vector_cosine_ops)
WITH (m = 16, ef_construction = 64);
```

### 第四阶段：全文获取集成 (1-2周)

#### 4.1 Google Patents集成（唯一外部数据源）

**架构调整说明**：
根据最新决策，系统已删除CNIPA检索器，现在Google Patents是唯一的外部专利全文获取来源。通过优化后的Google Patents服务，确保稳定可靠的全文获取。

```python
class OptimizedGooglePatentsService:
    """
    优化的Google Patents服务
    - 限制获取条数：每次最多20条专利
    - 多策略获取：元标签解析 + 结构化数据API + 浏览器自动化
    - 智能缓存：Redis缓存 + 内存缓存
    - 并发处理：支持多专利并发获取
    """

    def __init__(self):
        self.config = {
            'timeout': 15,
            'max_retries': 3,
            'retry_delay': 1.0,
            'batch_delay': 2.0,
            'max_concurrent': 5,
            'cache_ttl': 3600,
            'max_citations': 20,  # 最大引用数限制
            'max_family_members': 10,  # 最大专利族成员数
            'max_results': 20  # 每次搜索最大结果数
        }

    async def fetch_patent_fulltext(self, patent_id: str):
        # 策略1: 元标签解析（主要策略）
        result = await self.fetch_via_metatags(patent_id)
        if result:
            return result

        # 策略2: 结构化数据API
        result = await self.fetch_via_api(patent_id)
        if result:
            return result

        # 策略3: 浏览器自动化（兜底策略）
        result = await self.fetch_via_browser(patent_id)
        return result
```

**配置限制**：
- 每次搜索最多返回20条专利
- 每个专利最多提取20个引用专利
- 每个专利族最多分析10个成员
- 智能缓存机制减少重复请求

#### 4.2 缓存策略
- **Redis缓存**：24小时过期
- **分层存储**：热点数据优先缓存
- **预加载**：常用查询预加载

- **语义检索**：捕获概念相关的专利

### 3. 结果优化保障
- **去重机制**：避免重复结果
- **智能排序**：相关度最高的优先
- **多样性保证**：覆盖不同角度的专利
- **完整性验证**：确保关键信息完整

### 4. 质量监控保障
- **检索日志**：记录所有检索请求
