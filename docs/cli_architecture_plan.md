## 💾 存储架构方案

### 1. **向量数据库选择**

#### 候选方案对比
| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Chroma** | 开源免费，Python生态好 | 性能一般 | ⭐️⭐️⭐️ |
| **Pinecone** | 性能优秀，托管服务 | 费用较高 | ⭐️⭐️⭐️⭐️ |
| **Qdrant** | 性能优秀，开源免费 | 相对较新 | ⭐️⭐️⭐️⭐️⭐️ |
| **Weaviate** | GraphQL支持，生态好 | 学习成本高 | ⭐️⭐️⭐️ |

#### 推荐方案：**Qdrant**
- **优势**: 性能优秀，完全开源，Python友好
- **端口**: 6333
- **用途**: 存储文档向量，语义搜索

### 2. **知识图谱选择**

#### 候选方案对比
| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Neo4j** | 功能强大，生态完善 | 商用收费 | ⭐️⭐️⭐️⭐️ |
| **JanusGraph** | 开源可扩展 | 配置复杂 | ⭐️⭐️⭐️ |
| **NetworkX** | Python原生，简单易用 | 性能一般 | ⭐️⭐️⭐️⭐️⭐️ |
| **ArangoDB** | 多模型，文档+图 | 学习成本高 | ⭐️⭐️⭐️ |

#### 推荐方案：**NetworkX + PostgreSQL**
- **NetworkX**: 纯Python，简单易用，适合中小规模
- **PostgreSQL**: 扩展了图数据库功能
- **组合优势**: 简单易用，成本可控

### 3. **原文件存储方案**

#### 存储结构
```
/Volumes/AthenaData/
├── documents/           # 原始文档
│   ├── apps/patents/         # 专利文档
│   ├── contracts/       # 合同文档
│   ├── cases/           # 案例文档
│   └── modules/knowledge/knowledge/       # 知识文档
├── processed/          # 处理后的文档
│   ├── vectors/         # 向量化结果
│   ├── extracted/       # 提取的实体
│   └── summaries/       # 摘要信息
├── media/              # 多媒体文件
│   ├── images/          # 图片
│   ├── videos/          # 视频
│   └── audio/           # 音频
└── backups/            # 备份文件
```

#### 存储技术
- **本地存储**: 文件系统 + 符号链接
- **云存储**: 阿里云OSS / 腾讯云COS (可选)
- **版本控制**: Git + LFS (大文件)
