## 🔄 串行+并行执行策略

### 并行执行规则

**可以并行执行**:
- 不同智能体的不同任务
- 不相互依赖的工具
- 独立的文件和模块

**示例**:
```
智能体2: Bash工具 ┐────┐
智能体3: Read工具 ─────┤→ 并行执行
智能体4: Edit工具 ─────┘
```

### 串行执行规则

**必须串行执行**:
- 有依赖关系的任务
- 需要共享资源的任务
- 需要按顺序测试的工具

**示例**:
```
任务#7 (修复注册)
  ↓
任务#8 (P0基础工具)
  ↓
任务#9 (P2 Agent工具)
```
