## 🛠️ 技术实现

### 关键技术

1. **批量导入策略**
   - 批量大小: 1,000-10,000条
   - 事务控制: 自动提交
   - 进度监控: 实时反馈

2. **索引优化**
   - Entity.id唯一约束
   - Entity.text属性索引
   - OpenClawNode._original_id索引

3. **ID映射策略**
   - 本项目Entity: 使用PostgreSQL的id字段
   - OpenClawNode: 使用_original_id存储原始ID
   - 关系导入: 通过_original_id匹配节点

### 解决的问题

**问题1: 列名错误**
- 错误: column "entity_id" does not exist
- 解决: 使用正确的列名（id, entity_text等）

**问题2: 节点ID不匹配**
- 问题: 源数据库使用原始ID，目标数据库使用element_id
- 解决: 保留_original_id字段，使用原始ID匹配

**问题3: 关系导入速度慢**
- 问题: 通过文本匹配节点，效率低
- 解决: 创建Entity.text索引，速度提升至584条/秒

**问题4: 数据冲突**
- 问题: 两个数据集可能有重叠
- 解决: 使用不同标签和关系类型

```bash
# 关系统计
docker exec athena-neo4j-dev cypher-shell -u neo4j -p athena_neo4j_2024 \
  "MATCH ()-[r]->() RETURN type(r) as type, count(*) as count ORDER BY count DESC"

# 总统计
docker exec athena-neo4j-dev cypher-shell -u neo4j -p athena_neo4j_2024 \
  "MATCH (n) WITH count(n) as nodes MATCH ()-[r]->() RETURN nodes, count(r) as relationships"
```
