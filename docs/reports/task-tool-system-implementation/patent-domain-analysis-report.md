## 🎯 专利代理类型定义

### 1. 专利分析师 (patent-analyst)

**角色定位**: 专利技术分析专家

**核心能力**:
- 专利技术分析
- 创新点识别
- 技术对比分析
- 技术方案评估

**典型场景**:
1. **技术方案分析**: 分析专利申请的技术方案，识别核心技术特征
2. **创新点识别**: 从技术方案中识别创新点和技术贡献
3. **现有技术对比**: 将目标专利与现有技术进行对比分析
4. **技术价值评估**: 评估专利的技术价值和商业价值

**工具需求**:
- `code_analyzer`: 技术代码分析
- `knowledge_graph`: 知识图谱查询
- `patent_search`: 专利检索
- `web_search`: 网络搜索

**工作流需求**:
```
输入: 专利文本/技术方案
处理:
  1. 技术特征提取
  2. 创新点识别
  3. 现有技术检索
  4. 对比分析
  5. 报告生成
输出: 技术分析报告
```

---

### 2. 专利检索员 (patent-searcher)

**角色定位**: 专利检索策略专家

**核心能力**:
- 检索策略制定
- 关键词提取和扩展
- 分类号选择
- 检索式构建
- 对比文件筛选

**典型场景**:
1. **新颖性检索**: 检索破坏专利新颖性的对比文件
2. **创造性检索**: 检索破坏专利创造性的对比文件
3. **技术调查**: 了解特定技术领域的专利布局
4. **侵权检索**: 检索可能侵权的专利

**工具需求**:
- `patent_search`: 专利数据库检索
- `web_search`: 网络搜索
- `knowledge_graph`: 知识图谱查询

**工作流需求**:
```
输入: 技术方案/检索需求
处理:
  1. 需求分析
  2. 关键词提取
  3. 检索式构建
  4. 多源检索
  5. 结果筛选
  6. 报告生成
输出: 检索报告
```

---

### 3. 法律研究员 (legal-researcher)

**角色定位**: 专利法律研究专家

**核心能力**:
- 法律法规解读
- 案例分析
- 法律条文解读
- 风险评估

**典型场景**:
1. **法规研究**: 研究专利法及相关法规
2. **案例分析**: 分析典型案例和复审无效决定
3. **法律风险**: 评估专利的法律风险
4. **合规建议**: 提供专利合规建议

**工具需求**:
- `knowledge_graph`: 法律知识图谱
- `web_search`: 法规和案例搜索
- `patent_search`: 专利检索

**工作流需求**:
```
输入: 法律问题/案例
处理:
  1. 问题分析
  2. 法规检索
  3. 案例检索
  4. 法理分析
  5. 意见生成
输出: 法律意见书
```

---

### 4. 专利撰写员 (patent-drafter)

**角色定位**: 专利申请文件撰写专家

**核心能力**:
- 专利申请文件撰写
- 权利要求书撰写
- 说明书撰写
- 专利答复撰写

**典型场景**:
1. **申请文件撰写**: 撰写专利申请文件
2. **权利要求优化**: 优化权利要求书
3. **答复撰写**: 撰写审查意见答复
4. **文件修改**: 修改专利申请文件

**工具需求**:
- `document_processor`: 文档处理
- `code_analyzer`: 代码分析（技术方案）
- `knowledge_graph`: 知识图谱查询

**工作流需求**:
```
输入: 技术方案/审查意见
处理:
  1. 技术理解
  2. 文件结构设计
  3. 内容撰写
  4. 质量检查
  5. 输出生成
输出: 专利申请文件/答复文件
```

## 📊 集成需求

### 1. 与TaskTool的集成

**需求**:
- 所有工作流通过TaskTool执行
- 支持同步和异步执行
- 支持后台任务管理
- 支持任务恢复和取消

**实现**:
- 每个工作流封装为TaskTool任务
- 使用SubagentRegistry选择代理类型
- 使用ToolFilter过滤可用工具
- 使用ForkContextBuilder构建上下文

### 2. 与四层记忆系统的集成

**需求**:
- HOT层: 缓存检索结果、分析报告
- WARM层: 缓存专利数据、知识图谱
- COLD层: 持久化专利文件、分析历史
- ARCHIVE层: 归档旧版本文件

**实现**:
- 检索结果缓存到WARM层
- 分析报告存储到COLD层
- 长期数据归档到ARCHIVE层

### 3. 与知识图谱的集成

**需求**:
- 查询专利技术关系
- 查询法律知识图谱
- 查询案例关系图谱

**实现**:
- 通过knowledge_graph工具查询
- 支持SPARQL查询
- 支持图遍历算法
