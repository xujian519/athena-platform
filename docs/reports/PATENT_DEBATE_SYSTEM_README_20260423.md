## 🎯 核心技术

### 1. curl调用方案

**问题**: Python HTTP库（httpx、requests）与oMLX不兼容

**解决**: 使用subprocess调用curl

```python
def _call_omlx(self, user_message: str) -> str:
    payload = {
        "model": self.model,
        "messages": [
            {"role": "system", "content": self.system_prompt},
            {"role": "user", "content": user_message}
        ],
        "max_tokens": 800,
        "temperature": 0.8
    }

    curl_cmd = [
        "curl", "-s", "-X", "POST", self.api_url,
        "-H", "Content-Type: application/json",
        "-H", f"Authorization: Bearer {self.api_key}",
        "-d", json.dumps(payload),
        "--max-time", "120"
    ]

    result = subprocess.run(curl_cmd, capture_output=True, text=True, timeout=180)
    if result.returncode == 0:
        response = json.loads(result.stdout)
        return response["choices"][0]["message"]["content"]
```

**价值**: 避免Python HTTP库兼容性问题，实现本地模型完美集成

### 2. 上下文优化

**问题**: 3000+ tokens导致超时

**解决**: 优化到800 tokens，保留核心信息

