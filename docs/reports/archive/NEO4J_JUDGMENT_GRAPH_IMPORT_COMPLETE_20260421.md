### Neo4j索引

```cypher
-- Entity节点约束
CREATE CONSTRAINT entity_id IF NOT EXISTS
FOR (e:Entity) REQUIRE e.id IS UNIQUE

-- Entity.text属性索引（用于关系匹配）
CREATE INDEX entity_text_index FOR (e:Entity) ON (e.text)
```

## 🔧 技术挑战与解决

### 挑战1: 关系导入速度慢

**问题**:
- 最初使用文本匹配，在89万个节点中查找极慢
- 测试1000条关系耗时数分钟

**解决方案**:
- 创建`Entity.text`属性索引
- 使用索引查询，速度提升至584条/秒

### 挑战2: 数据结构不匹配

**问题**:
- `judgment_relations`表使用文本而非ID引用实体
- 需要通过文本匹配找到对应节点

**解决方案**:
- 使用Entity.text属性索引
- 批量查询优化

### 挑战3: Neo4j全文索引未启用

**问题**:
- `db.index.fulltext`过程未注册
- 无法使用全文搜索

**解决方案**:
