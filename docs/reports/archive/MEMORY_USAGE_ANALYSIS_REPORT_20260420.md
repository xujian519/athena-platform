## 🔍 macOS内存管理机制

### 1. 内存压缩（Compressed Memory）

macOS使用**内存压缩技术**来优化内存使用：

```
PhysMem: 47G used (3987M wired, 22G compressor), 434M unused
```

**解读**:
- **47GB used**: 不全是应用占用的
- **22GB compressor**: 这是macOS自动压缩的不活跃内存
- **3987MB wired**: 系统核心内存（无法被换出）
- **434MB unused**: 真正空闲的内存

### 2. 真实应用内存计算

```
真实应用内存 = 总内存 - 压缩内存 - 空闲内存 - wired内存
```

根据当前数据：
- 系统总内存: ~48GB (估算)
- 压缩内存: 22GB
- 空闲内存: 434MB
- Wired内存: 3987MB
- **真实应用内存: ~21.5GB (约45%)**

### 3. macOS内存优先级

macOS按以下优先级管理内存：

1. **Wired** (最高): 系统核心，无法换出
2. **Active**: 正在使用的应用内存
3. **Inactive**: 最近使用过但现在不活跃
4. **Compressed**: 被压缩的不活跃内存（可快速解压）
5. **Free** (最低): 完全空闲

## 📈 监控建议

### 关键指标监控

使用以下命令监控系统健康：

```bash
# 查看真实内存压力
vm_stat | grep Pages

# 查看Swap使用
sysctl vm.swapusage

# 查看页面换入/换出
vm_stat | grep "Pageins\|Pageouts"
```

### 告警阈值

只有在以下情况才需要关注内存：

1. **Swap使用 > 1GB**: 系统有内存压力
2. **Pageouts持续增长**: 频繁换出到磁盘
3. **应用响应变慢**: 内存压力影响性能
