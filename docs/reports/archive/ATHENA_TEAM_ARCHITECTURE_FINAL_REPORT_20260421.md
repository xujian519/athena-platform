## 📊 项目概览

### 核心目标

将原有的"小娜单智能体"拆分为多个专业智能体（Athena团队），通过"小诺"作为主智能体进行动态编排，实现：

1. **专业化分工**：每个智能体专注于特定领域
2. **人机协同**：用户作为决策者，智能体作为执行者
3. **先规划再执行**：小诺0号原则，执行前必须展示计划并获得确认
4. **质量优先**：专利法律业务以质量为最高原则

### 实施策略

- ✅ **渐进式拆分**: 分3个阶段逐步实现
- ✅ **向后兼容**: 保留小娜统一入口
- ✅ **动态编排**: 小诺根据场景自动组装智能体
- ✅ **易于扩展**: 新增智能体只需注册

## 🎯 核心设计亮点

### 1. 渐进式架构演进

```
Phase 1 (当前): 4个基础智能体
    ↓
Phase 2: +2个审查智能体
    ↓
Phase 3: +1个无效宣告智能体
```

**优势**:
- 每个阶段都可以独立运行和验证
- 降低重构风险
- 便于A/B测试和数据驱动决策

### 2. 小诺0号原则

**定义**: 任何复杂任务在执行前必须先制定执行计划，并向用户展示，获得确认后才能执行。

**实施方式**:
- 场景识别后，立即制定执行计划
- 展示执行模式、步骤、调用的智能体、预估时间
- 用户确认后执行
- 执行过程中允许用户打断和调整

### 3. 双格式输出

**定义**: 每个智能体必须同时输出JSON格式（机器可读）和Markdown格式（人类可读）。

**实施方式**:
```json
{
  "structured_data": { ... },  // JSON格式，机器可读
  "markdown_text": "..."       // Markdown格式，人类可读
}
```

### 4. 智能体职责清晰划分

**纯技术分析**:
- 分析者（AnalyzerAgent）：技术特征提取、三原则提取、技术总结

**技术+法律分析**:
- 创造性分析智能体：三步法判断
- 新颖性分析智能体：单独对比原则判断
- 侵权分析智能体：全面覆盖+等同原则判断

**形式+实质审查**:
- 申请文件审查智能体：权利要求+说明书分析
- 撰写审查智能体：撰写质量审查

**高级法律分析**:
- 无效宣告分析智能体：无效宣告请求/答辩

### 5. 四种执行模式

**串行（Sequential）**:
- 步骤按顺序依次执行
- 适用：简单任务、强依赖关系

**并行（Parallel）**:
- 多个步骤同时执行
- 适用：多数据源检索、多专利同时分析

**迭代（Iterative）**:
- 某个步骤循环执行直到满足条件
- 适用：迭代式搜索、多轮分析

**混合（Hybrid）**:
- 组合多种执行模式
- 适用：复杂任务

### 6. 完善的人机协作机制

**控制按钮**:
- 暂停、继续、查看进度、调整计划、停止

**打断时机**:
- 步骤之间（默认）
- 步骤内部（关键任务）
- 多轮迭代（用户指示）

**调整自由度**:
- 添加/删除步骤
- 调整智能体组合
- 修改执行模式
- 改变输入参数
