## 🚀 快速开始

### 方式1: Python代码调用

```python
import asyncio
from core.perception.technical_drawing_analyzer import (
    TechnicalDrawingAnalyzer,
    AnalysisLevel
)
from services.ai_models.glm_full_suite.glm_unified_client import ZhipuAIUnifiedClient

async def main():
    # 1. 初始化客户端
    async with ZhipuAIUnifiedClient() as glm_client:
        # 2. 创建分析器
        analyzer = TechnicalDrawingAnalyzer(
            glm_client=glm_client,
            max_image_size=2048,    # 最大图像尺寸
            chunk_size=1024,        # 分块大小
            enable_chunking=True    # 启用分块处理
        )

        # 3. 分析图纸
        result = await analyzer.analyze(
            image_path="path/to/drawing.jpg",
            analysis_level=AnalysisLevel.INTERMEDIATE,
            specification_text="说明书文本(可选)",
            related_claims=["权利要求1", "权利要求2"]
        )

        # 4. 查看结果
        print(f"图纸类型: {result.drawing_type}")
        print(f"标题: {result.title}")
        print(f"描述: {result.description}")
        print(f"组件数量: {len(result.components)}")
        print(f"技术特征: {result.technical_features}")

asyncio.run(main())
```

## 🔧 配置参数

### TechnicalDrawingAnalyzer 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `glm_client` | ZhipuAIUnifiedClient | 必需 | GLM统一客户端 |

### 分析级别 (AnalysisLevel)

| 级别 | 说明 | Token消耗 | 适用场景 |
|------|------|----------|----------|
| `BASIC` | 基础识别 | ~500 | 快速查看图纸内容 |
| `INTERMEDIATE` | 中级分析 | ~1500 | 提取组件和特征 |
| `ADVANCED` | 高级分析 | ~3000 | 深度理解和关联 |

## 🔍 分块处理机制

### 何时自动分块?

当图像尺寸超过 `chunk_size` (默认1024像素)时,系统会自动启用分块处理:

```python
# 2048x2048的图像会分成4块
# ┌─────┬─────┐
# │  1  │  2  │
# ├─────┼─────┤
# │  3  │  4  │
# └─────┴─────┘
```

### 分块处理流程

1. **图像分割**: 将大图均匀分成多个小块
2. **逐块分析**: 使用GLM-4V分析每块内容
3. **结果合并**: 智能合并所有分块的结果
4. **整体理解**: 基于所有分块生成整体理解

### 优势

- ✅ **解决上下文限制**: 避免单次请求超出token限制
- ✅ **提高准确度**: 分块后细节识别更准确
- ✅ **并行处理**: 支持并发分析多个分块

## 🛡️ 上下文压缩机制

系统自动集成了上下文压缩器,避免token超出限制:

```python
# 上下文压缩器配置
context_compressor = ContextCompressor(

```

### 压缩策略

1. **智能压缩**: 保留重要消息,删除冗余内容
2. **时间窗口**: 优先保留最近的对话
3. **重要性评分**: 根据关键词和内容评估重要性

## 🐛 故障排查

### 问题1: "context window limit" 错误

**原因**: 图像太大或对话历史太长

**解决方案**:
```python
# 启用分块处理
analyzer = TechnicalDrawingAnalyzer(
    glm_client=glm_client,
    enable_chunking=True,
    chunk_size=512  # 减小分块大小
)
```

### 问题2: 识别准确度低

**原因**: 分析级别太低

**解决方案**:
```python
# 提高分析级别
result = await analyzer.analyze(
    image_path="complex_drawing.jpg",
    analysis_level=AnalysisLevel.ADVANCED  # 高级分析
)
```

### 问题3: 处理速度慢

**原因**: 分块太多或图像太大

**解决方案**:
```python
# 预处理图像:调整大小
analyzer = TechnicalDrawingAnalyzer(
    glm_client=glm_client,
    max_image_size=1024,  # 降低最大尺寸
    chunk_size=1024
)
```

### 与专利分析系统集成

```python
from core.perception.technical_drawing_analyzer import TechnicalDrawingAnalyzer

class PatentAnalysisSystem:
    def __init__(self):
        self.drawing_analyzer = TechnicalDrawingAnalyzer(...)
        # ... 其他组件

    async def analyze_patent(self, patent_data):
        # 分析权利要求
        claims_analysis = await self.analyze_claims(patent_data['claims'])

        # 分析说明书附图
        drawing_results = []
        for drawing in patent_data['drawings']:
            result = await self.drawing_analyzer.analyze(
                image_path=drawing['path'],
                analysis_level=AnalysisLevel.ADVANCED,
                specification_text=patent_data['specification'],
                related_claims=patent_data['claims']
            )
            drawing_results.append(result)

        # 综合分析
        return self.merge_analysis(claims_analysis, drawing_results)
```
