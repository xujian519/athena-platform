## 端口分配原则

### 核心原则

1. **内网通信优先**: 所有服务间通信必须通过 `127.0.0.1` 进行
2. **单一入口**: 外部访问只能通过 Gateway (8005)
3. **端口分段管理**: 按服务类型划分端口范围
4. **避免冲突**: 新服务分配前必须检查端口占用

### 绑定地址规范

| 服务类型 | 绑定地址 | 说明 |
|---------|---------|------|
| Gateway | `0.0.0.0:8005` | 唯一对外暴露的服务 |
| 所有内部服务 | `127.0.0.1` | 只接受本地连接 |
| 数据库服务 | `127.0.0.1` | 通过Docker网络访问 |

## 安全规范

### 访问控制

```
┌─────────────────────────────────────────────────────────────┐
│  外部请求                                                     │
│     │                                                       │
│     ▼                                                       │
│  ┌─────────┐                                               │
│  │Gateway  │  0.0.0.0:8005 (唯一对外暴露)                     │
│  │ 8005    │                                               │
│  └────┬────┘                                               │
│       │ 路由转发                                            │
│       ▼                                                    │
│  ┌─────────────────────────────────────────────────────┐  │
│  │          内网服务 (127.0.0.1)                         │  │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │  │
│  │  │ 小娜    │ │ 小诺    │ │ 云熙    │ │ 其他服务 │   │  │
│  │  │ 8101    │ │ 8102    │ │ 8103    │ │ ...     │   │  │
│  │  └─────────┘ └─────────┘ └─────────┘ └─────────┘   │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 配置规范

**所有内部服务必须使用以下配置**:

```python
# 正确 ✅
uvicorn.run(app, host="127.0.0.1", port=8101)

# 错误 ❌
uvicorn.run(app, host="0.0.0.0", port=8101)
```

```go
// 正确 ✅
server := &http.Server{
    Addr: "127.0.0.1:8101",
}

// 错误 ❌
server := &http.Server{
    Addr: ":8101",  // 默认绑定0.0.0.0
}
```

### 验证命令

```bash
# 检查服务是否只监听127.0.0.1
lsof -i -P -n | grep LISTEN | grep -v "127.0.0.1"

# 应该只看到Gateway 8005绑定到*
# 其他服务应该都显示127.0.0.1
```

## 端口管理工具

### port_manager.py

```bash
# 检查所有端口状态
python3 tools/port_manager.py check

# 为新服务分配端口
python3 tools/port_manager.py allocate <service_name>

# 生成端口报告
python3 tools/port_manager.py report
```

### 快速检查脚本

```bash
# 检查端口占用
lsof -i :<port>

# 查找所有Athena服务端口
lsof -i -P -n | grep -E "(LISTEN|python|athena)" | grep LISTEN

# 检查特定服务
ps aux | grep <service_name> | grep -v grep
```

## 故障排查

### 端口冲突

**症状**: 服务启动失败，提示端口已被占用

**解决步骤**:
1. 检查端口占用: `lsof -i :<port>`
2. 确认占用进程: `ps aux | grep <pid>`
3. 停止冲突进程或更换端口

### 无法访问服务

**症状**: 服务运行中但无法访问

**检查步骤**:
1. 确认服务监听地址: `lsof -i :<port>`
2. 如果是内部服务，确认通过Gateway访问
3. 检查Gateway路由配置

### Gateway无法转发

**症状**: Gateway正常但无法访问内部服务

**检查步骤**:
1. 确认内部服务监听 `127.0.0.1`
2. 检查Gateway路由配置
3. 查看Gateway日志: `docker-compose logs -f gateway`
