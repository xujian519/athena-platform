## 🎯 核心原则

### 优秀提示词的5个要素

```
┌─────────────────────────────────────┐
│     优秀提示词 = 角色 + 上下文       │
│                  + 任务 + 格式      │
│                  + 约束             │
└─────────────────────────────────────┘
```

| 要素 | 作用 | 示例 |
|------|------|------|
| **角色** | 设定AI身份 | "你是一位资深架构师" |
| **上下文** | 提供背景信息 | "项目是Python，使用PostgreSQL" |
| **任务** | 明确要做什么 | "分析这段代码的性能问题" |
| **格式** | 指定输出格式 | "请按：问题→原因→方案格式回答" |
| **约束** | 说明限制条件 | "不要使用外部库，保持PEP 8" |

## 💡 立即使用的3个技巧

### 技巧1: "让我们逐步思考"

**简单添加即可大幅提升复杂问题的解决质量**

```markdown
# 优化前
这个架构有什么问题？

# 优化后
这个架构有什么问题？

让我们逐步思考：
1. 先分析整体架构
2. 再逐层分析各模块
3. 识别潜在问题
4. 提出改进建议
```

**效果**: +40% 复杂问题解决能力

### 技巧2: 添加项目上下文

**在提示词开头引用项目上下文**

```markdown
# 在每次Claude Code会话开始时运行
请先阅读 .claude-project-context.md 了解项目背景

# 然后提出您的问题
现在请帮我分析...
```

**效果**: +30% 相关性和准确性

### 技巧3: 使用"角色扮演"

**让AI以特定角色视角回答**

```markdown
# 优化前
帮我评审这段代码

# 优化后
你是一位资深代码审查员，专注于：
- 代码质量和可维护性
- 潜在bug和边界条件
- 性能优化机会

请审查以下代码：
```

**效果**: +20% 专业性和深度

## 🛠️ 快速模板库

### 模板1: 代码分析

```markdown
# 代码分析

## 代码
\`\`\`python
{您的代码}
\`\`\`

## 分析要求
1. 功能: 这段代码做什么？
2. 质量: 代码质量如何？
3. 问题: 有什么潜在问题？
4. 优化: 如何改进？

请提供具体的改进建议和代码示例。
```

### 模板2: Bug诊断

```markdown
# Bug 诊断

## 问题描述
{bug描述}

## 相关代码
\`\`\`python
{代码片段}
\`\`\`

## 诊断要求
1. 问题定位: 哪里出错了？
2. 根因分析: 为什么会出错？
3. 解决方案: 如何修复？
4. 预防: 如何避免再次发生？

请提供可执行的修复代码。
```

### 模板3: 架构设计

```markdown
# 架构设计

## 需求
{功能需求}

## 设计要求
请以资深架构师视角设计：
1. 整体架构图
2. 模块划分
3. 接口设计
4. 数据流
5. 扩展性考虑

## 约束
- 技术栈: Python + PostgreSQL + Redis
- 性能: 支持100+ QPS
- 可用性: 99.9%+

请提供详细的设计方案。
```

## 🎓 进阶技巧

### 技巧: 迭代优化

```markdown
# 第1轮: 初步分析
{基础提示词}

# 第2轮: 深化分析
基于上面的分析，请进一步：
- 补充性能数据
- 提供具体实现方案

# 第3轮: 完善
请基于前两轮，提供：
1. 完整实现代码
2. 测试用例
3. 文档说明
```

### 技巧: 多角色协同

```markdown
# 让不同角色协同分析

请让以下角色分别分析：

后端架构师:
- 可扩展性评估
- 接口设计建议

DevOps工程师:
- 部署方案
- 监控指标

安全专家:
- 安全风险评估
- 缓解措施

最后综合各方观点给出建议。
```
