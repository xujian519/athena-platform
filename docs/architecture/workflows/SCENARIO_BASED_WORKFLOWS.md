## 📋 文档概述

本文档定义Athena团队在各种业务场景下的工作流程，包括场景识别、智能体组装、执行模式、人机协作节点等。

---

## 🎯 核心原则

### 1. 先规划再执行（小诺0号原则）

**定义**：任何复杂任务在执行前必须先制定执行计划，并向用户展示，获得确认后才能执行。

**实施方式**：
- 场景识别后，立即制定执行计划
- 展示执行模式、步骤、调用的智能体、预估时间
- 用户确认后执行
- 执行过程中允许用户打断和调整

### 2. 质量优先原则

**定义**：专利和法律业务以质量为最高原则，性能和成本服从于质量。

**实施方式**：
- 不追求速度而牺牲准确性
- 不追求低成本而影响专业性
- 确保输出的专业性和准确性
- 完整的双格式输出（JSON + Markdown）

## 🔄 场景工作流程

### 场景1：专利检索（PATENT_SEARCH）

**触发条件**：
- 用户明确要求检索专利
- 关键词：检索、搜索、查找专利、similar documents

**智能体组装**：
```
小诺 → 检索者 → 结果汇总
```

**执行模式**：串行（Sequential）

**工作流程**：
```
1. 用户输入："帮我检索关于自动驾驶掉头的专利"

2. 小诺场景识别
   - 识别场景：PATENT_SEARCH
   - 确定智能体：检索者
   - 制定执行计划

3. 展示执行计划
   - 执行模式：串行
   - 步骤数：1
   - 智能体：检索者
   - 预估时间：15秒

4. 用户确认（自动/手动）

5. 执行检索
   - 检索者：
     a. 关键词扩展
     b. 多数据库检索
     c. 结果筛选和排序
     d. 下载全文（如果需要）

6. 结果汇总
   - JSON格式：专利列表（公开号、标题、摘要）
   - Markdown格式：检索报告

7. 输出结果
```

**人机协作节点**：
- ❌ 无需确认（简单场景）
- ✅ 可执行中打断（查看进度、调整策略）

**输出格式**：
```json
{
  "scenario": "patent_search",
  "output": {
    "keywords": ["自动驾驶", "掉头", "路径规划"],
    "patents": [
      {
        "patent_id": "CN123456789A",
        "publication_number": "CN123456789A",
        "title": "自动驾驶车辆掉头方法",
        "abstract": "本发明公开了...",
        "url": "https://patents.google.com/patent/CN123456789A"
      }
    ],
    "total_count": 50,
    "filtered_count": 20
  }
}
```

### 场景3：创造性分析（CREATIVITY_ANALYSIS）

**触发条件**：
- 用户要求分析专利创造性
- 关键词：创造性、三步法、技术启示、obviousness

**智能体组装**：
```
小诺 → 规划者（Phase 2）→ 检索者 → 分析者 → 创造性分析智能体 → 结果汇总
```

**执行模式**：混合（Hybrid）

**工作流程**：
```
1. 用户输入："分析专利CN123456789A的创造性"

2. 小诺场景识别
   - 识别场景：CREATIVITY_ANALYSIS
   - 确定智能体：规划者（Phase 2）→ 检索者 → 分析者 → 创造性分析智能体
   - 制定执行计划

3. 展示执行计划（复杂场景，必须展示）
   - 执行模式：混合
   - 步骤数：4
   - 智能体：
     1. 规划者：任务拆解、策略制定
     2. 检索者：检索对比文件
     3. 分析者：技术特征提取
     4. 创造性分析智能体：三步法分析
   - 预估时间：60秒

4. 用户确认（必须手动确认）

5. 执行分析
   - 步骤1：规划者
     a. 任务拆解
     b. 检索策略制定
     c. 对比文件选择标准

   - 步骤2：检索者
     a. 关键词扩展
     b. 检索对比文件（D1、D2、D3...）
     c. 下载全文

   - 步骤3：分析者
     a. 目标专利技术特征提取
     b. 对比文件技术特征提取
     c. 特征对比分析

   - 步骤4：创造性分析智能体
     a. 最接近的现有技术确定（Step 1）
     b. 区别特征识别（Step 2）
     c. 技术问题确定（Step 2）
     d. 技术启示判断（Step 3）
     e. 辅助判断因素分析
     f. 创造性结论

6. 结果汇总
   - JSON格式：三步法分析结构化数据
   - Markdown格式：创造性分析报告

7. 输出结果
```

**人机协作节点**：
- ✅ 执行前必须确认（复杂场景）
- ✅ 可执行中打断（查看进度、调整对比文件、深入分析）
- ✅ 支持迭代（用户指示：深入分析某个对比文件）

**输出格式**：
```json
{
  "scenario": "creativity_analysis",
  "output": {
    "three_step_analysis": {
      "step1_closest_prior_art": "D1：CN123456788A，名称为...",
      "step2_distinctive_features": [
        "特征1：使用拟人化驾驶行为模型",
        "特征2：动态调整掉头轨迹"
      ],
      "step2_technical_problem": "实际解决的技术问题：如何提高自动驾驶车辆在狭窄空间的掉头安全性和效率",
      "step3_obviousness": false,
      "step3_technical_teaching": "D1、D2、D3均无明确启示"
    },
    "secondary_considerations": {
      "unexpected_effect": "掉头成功率提升30%，超出预期",
      "technical_prejudge": "无",
      "commercial_success": "已在多个车型应用"
    },
    "creativity_conclusion": "具备创造性",
    "creativity_level": "高",
    "reasoning": "基于三步法分析，区别特征非显而易见，且有预料不到的技术效果"
```

## 🎮 人机协作机制

### 控制按钮

前端提供以下控制按钮：
- **暂停**：暂停当前执行，保留状态
- **继续**：继续执行
- **查看进度**：查看当前执行进度和中间结果
- **调整计划**：调整后续执行计划
- **停止**：停止执行，清空状态

### 打断时机

**可打断时机**：
- ✅ 步骤之间（默认）
- ✅ 步骤内部（关键任务）
- ✅ 多轮迭代（用户指示）

**不可打断时机**：
- ❌ 单个LLM调用内部（原子操作）

### 调整自由度

**高度自由**：
- 添加/删除步骤
- 调整智能体组合
- 修改执行模式
- 改变输入参数

## 📈 执行模式详解

### 1. 串行（Sequential）

**定义**：步骤按顺序依次执行

**适用场景**：
- 简单任务
- 步骤间有强依赖关系

**示例**：
```
步骤1 → 步骤2 → 步骤3
```

### 2. 并行（Parallel）

**定义**：多个步骤同时执行

**适用场景**：
- 多数据源检索
- 多专利同时分析
- 多策略同时尝试

**示例**：
```
步骤1 ─┬→ 步骤2a
       ├→ 步骤2b
       └→ 步骤2c
        ↓
       步骤3（汇总）
```

### 3. 迭代（Iterative）

**定义**：某个步骤循环执行直到满足条件

**适用场景**：
- 迭代式搜索
- 多轮分析（用户指示）
- 关键任务需要深度分析

**示例**：
```
步骤1 → 步骤2 ←┐
       ↑        │
       └────────┘
     （条件满足则跳出）
```

### 4. 混合（Hybrid）

**定义**：组合多种执行模式

**适用场景**：
- 复杂任务
- 需要灵活控制

**示例**：
```
步骤1 → 步骤2（并行：2a、2b、2c）→ 步骤3（迭代）→ 步骤4
```
