## 📋 文档概述

本文档定义Athena团队的人机交互机制，包括计划展示、用户确认、打断机制、调整自由度等。

---

## 🎯 核心原则

### 1. 先规划再执行（小诺0号原则）

**定义**：任何复杂任务在执行前必须先制定执行计划，并向用户展示，获得确认后才能执行。

**实施方式**：
- 场景识别后，立即制定执行计划
- 展示执行模式、步骤、调用的智能体、预估时间
- 用户确认后执行
- 执行过程中允许用户打断和调整

### 2. 人机协同原则

**定义**：用户是执行复杂任务的决策者，智能体是执行者和建议者。

**实施方式**：
- 关键节点必须用户确认
- 允许用户随时打断
- 允许用户高度自由的调整计划
- 智能体提供建议，用户做决策

### 3. 质量优先原则

**定义**：专利和法律业务以质量为最高原则，性能和成本服从于质量。

**实施方式**：
- 不追求速度而牺牲准确性
- 不追求低成本而影响专业性
- 确保输出的专业性和准确性

## 📊 计划展示

### 触发条件

**必须展示计划的场景**（复杂场景）：
- ✅ 正式法律分析：创造性分析、新颖性分析、侵权分析、无效宣告
- ✅ 专利撰写
- ✅ 审查意见答复
- ✅ 复审请求/答辩

**无需展示计划的场景**（简单场景）：
- ❌ 简单检索
- ❌ 简单技术分析
- ❌ 状态查询

**展示要素**：

### 展示时机

**策略A：立即展示（复杂场景）**
- 适用：创造性分析、专利撰写等正式工作
- 方式：场景识别后立即展示计划

**策略B：先分析后展示（中等复杂度）**
- 适用：检索、技术分析
- 方式：快速分析后再展示计划

**策略C：无需展示（简单场景）**
- 适用：状态查询、简单问答
- 方式：直接执行

## ✅ 用户确认

### 确认方式

**方式1：自动确认（简单场景）**
- 适用：检索、简单分析
- 方式：展示计划后3秒内无操作则自动执行

**方式2：手动确认（复杂场景）**
- 适用：正式法律分析、专利撰写
- 方式：必须用户点击"确认"按钮

**方式3：交互式选择（高复杂度）**
- 适用：需要用户选择的场景
- 方式：提供选项让用户选择

## 🎮 控制按钮

### 按钮定义

| 按钮 | 功能 | 触发时机 | 状态影响 |
|------|------|---------|---------|
| **暂停** | 暂停当前执行 | 执行中 | 保留状态 |
| **继续** | 继续执行 | 暂停后 | 恢复状态 |
| **查看进度** | 查看当前进度和中间结果 | 执行中 | 无影响 |
| **调整计划** | 调整后续执行计划 | 步骤之间 | 修改后续步骤 |
| **停止** | 停止执行，清空状态 | 执行中 | 清空状态 |

### 按钮状态

**执行中的按钮状态**：
```
[暂停] [查看进度] [调整计划] [停止]
```

**暂停后的按钮状态**：
```
[继续] [查看进度] [停止]
```

**步骤之间的按钮状态**：
```
[继续] [调整计划] [停止]
```

## 🔀 打断机制

### 可打断时机

**默认可打断**：
- ✅ 步骤之间（默认）
- ✅ 关键任务步骤内部（检索、分析）

**不可打断**：
- ❌ 单个LLM调用内部（原子操作）

### 打断方式

**方式1：暂停按钮**
- 用户点击"暂停"按钮
- 当前步骤完成后暂停
- 保留中间状态

**方式2：调整计划**
- 用户点击"调整计划"按钮
- 当前步骤完成后进入调整界面
- 可修改后续步骤

**方式3：用户指示**
- 用户发送新的指示
- 当前步骤完成后处理新指示
- 可能触发迭代

### 打断后的处理

**暂停后**：
```json
{
  "status": "paused",
  "current_step": "step_2",
  "completed_steps": ["step_1"],
  "remaining_steps": ["step_3", "step_4"],
  "intermediate_results": {
    "step_1": { ... }
  },
  "can_resume": true
}
```

**调整计划后**：
```json
{
  "status": "adjusted",
  "original_plan": { ... },
  "adjusted_plan": {
    "steps": ["step_1", "step_2", "step_3_new", "step_4_new"],
    "total_steps": 4,
    "estimated_time": 80
  },
  "changes": [
    "删除：step_3（技术特征提取）",
    "新增：step_3_new（深度技术分析）",
    "修改：step_4_new（加强创造性分析）"
  ]
}
```

## 🔄 迭代机制

### 迭代触发条件

**用户明确指示**：
- "深入分析D1"
- "再检索一些相关专利"
- "重新评估创造性"

**关键任务自动迭代**：
- 迭代式搜索
- 多轮分析
- 质量检查

### 迭代实现

**迭代式搜索**：
```python
async def iterative_search(agent, query, max_iterations=3):
    """迭代式搜索"""
    results = []
    for i in range(max_iterations):
        # 执行搜索
        new_results = await agent.search(query)
        results.extend(new_results)

        # 检查是否满足条件
        if len(results) >= 20 or i == max_iterations - 1:
            break

        # 根据结果调整查询
        query = agent.refine_query(query, new_results)

    return results
```

**多轮分析**：
```python
async def multi_round_analysis(agent, document, max_rounds=3):
    """多轮分析"""
    analysis = None
    for i in range(max_rounds):
        # 执行分析
        analysis = await agent.analyze(document, previous_analysis=analysis)

        # 检查是否满足深度要求
        if analysis.depth >= agent.required_depth:
            break

    return analysis
```

## 🎨 交互建议

### 小诺的建议

**建议时机**：
- 用户不确定如何选择时
- 有多种可行方案时
- 需要权衡利弊时

**建议格式**：
```
小诺：我建议选择"深入分析"模式，理由如下：
1. 这是正式的创造性分析任务，需要更深入的分析
2. 深入模式会检索更多对比文件，提高分析准确性
3. 预计时间增加20秒，但质量提升显著

您是否采纳这个建议？[采纳] [不采纳，使用标准模式]
```

### 智能体的建议

**检索者的建议**：
```
检索者：根据您的关键词，我建议：
1. 使用中英文双语检索，覆盖更全面
2. 优先检索CNIPA和Google Patents
3. 限制最近5年的专利

是否采纳？[采纳] [调整]
```

**分析者的建议**：
```
分析者：技术特征较多，我建议：
1. 先提取必要特征，再分析附加特征
2. 使用三原则框架组织分析

是否采纳？[采纳] [调整]
```
