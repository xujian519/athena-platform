## 📋 文档概述

本文档说明Athena团队架构如何与现有的成熟工作流（专利撰写、审查意见答复）无缝整合，保留现有优势，同时引入专业化分工的改进。

---

## 🎯 整合核心原则

### 1. 保留现有优势

**现有工作流的成熟优势**：
- ✅ **完整的流程定义**：5步撰写流程、5步答复流程
- ✅ **核心模块完善**：AutoSpecDrafter、SmartOAResponder等
- ✅ **质量评估体系**：7维度评估、赫布学习优化
- ✅ **人机协作协议**：详细的交互点和确认机制

**整合原则**：
- **不破坏现有流程**：保留5步结构，保留质量检查
- **不降低质量**：保留7维度评估、赫布学习
- **不增加复杂度**：用户感知不到架构变化

### 2. 引入专业化改进

**Athena团队的改进优势**：
- ✅ **智能体职责清晰**：每个智能体专注特定领域
- ✅ **知识来源明确**：基于宝宸知识库构建
- ✅ **动态编排灵活**：小诺根据场景组装智能体
- ✅ **易于扩展维护**：新增智能体只需注册

**整合策略**：
- **智能体映射到步骤**：将现有模块封装为智能体
- **保留核心模块**：现有代码复用，仅添加编排层
- **渐进式迁移**：先并行运行，验证后切换

## 🔄 整合工作流示例

### 示例1：专利撰写（整合后）

**用户输入**："帮我根据技术交底书撰写专利申请文件"

**执行流程**：

```
1. 小诺场景识别
   - 识别场景：PATENT_DRAFTING（专利撰写）
   - 确定智能体：检索者 → 分析者 → 撰写者 → 审查智能体（Phase 2）
   - 制定执行计划

2. 展示执行计划
   - 执行模式：混合（检索并行，其他串行）
   - 步骤数：5（对应现有5步骤）
   - 智能体：
     1. 分析者：技术交底书理解（AutoSpecDrafter封装）
     2. 检索者：现有技术检索（MultimodalRetrieval封装）
     3. 撰写者：说明书撰写（AutoSpecDrafter封装）
     4. 撰写者：权利要求撰写（PatentClaimGenerator封装）
     5. 撰写者：摘要撰写（XiaonaPatentDrafter封装）
   - 预估时间：总计约300秒（5分钟）
   - 质量保证：7维度评估，赫布学习优化

3. 用户确认（必须手动确认）

4. 执行撰写
   - 步骤1：分析者._understand_invention()
     a. 技术领域识别
     b. 技术问题提取
     c. 技术方案解析
     d. 技术效果分析
     e. 输出：InventionUnderstanding（保留数据结构）

   - 步骤2：检索者.semantic_search()
     a. 专利数据库检索
     b. 论文数据库检索
     c. 网络资源检索
     d. 输出：对比分析报告（保留数据结构）

   - 步骤3：撰写者.draft_specification()
     a. 技术领域撰写（50-100字）
     b. 背景技术撰写（300-500字）
     c. 发明内容撰写（800-1500字）
     d. 具体实施方式撰写（1500-3000字）
     e. 输出：SpecificationDraft（保留数据结构）

   - 步骤4：撰写者.generate_claims()
     a. 权利要求布局规划
     b. 独立权利要求撰写
     c. 从属权利要求撰写
     d. 输出：ClaimsSet（保留数据结构）

   - 步骤5：撰写者.write_abstract()
     a. 提取技术要点
     b. 撰写摘要（300字左右）
     c. 输出：专利申请文件_完整版.md

5. 质量检查（Phase 2智能体）
   - 申请文件审查智能体.quality_check()
   - 7维度评估：完整性、清晰性、准确性、充分性、一致性、规范性、支持性
   - 迭代优化（如果得分<7.5）

6. 结果汇总
   - JSON格式：结构化数据（保留现有数据结构）
   - Markdown格式：专利申请文件（保留现有格式）
   - 质量报告：7维度得分 + 总评
```

## 📈 迁移路径

### Phase 0：并行运行阶段（1-2周）

**目标**：验证Athena团队架构，不破坏现有功能

**实施方案**：
1. 保留现有工作流代码（完全不动）
2. 实现Athena团队架构（新增代码）
3. 封装现有模块为智能体（添加适配层）
4. 两个系统并行运行
5. 对比输出结果和质量

**验证指标**：
- 输出一致性：>95%
- 质量得分：不低于现有系统
- 执行时间：不慢于现有系统20%

### Phase 1：渐进式切换阶段（2-4周）

**目标**：逐步将用户流量切换到Athena团队架构

**实施方案**：
1. 用户可选择使用新旧系统
2. 收集用户反馈
3. 优化Athena团队架构
4. 逐步提高切换比例（10% → 50% → 100%）

**切换策略**：
- 简单任务优先：先切换简单撰写、简单答复
- 复杂任务后置：后切换复杂撰写、疑难答复
- 保留回滚机制：随时可以切回旧系统

### Phase 2：全面迁移阶段（4-6周）

**目标**：完全切换到Athena团队架构，废弃旧代码

**实施方案**：
1. 100%流量切换到Athena团队架构
2. 保留旧代码作为备份（1个月）
3. 移除旧代码（确认稳定后）
4. 清理技术债务

**废弃清单**：
- 旧工作流代码（标记为deprecated）
- 旧的提示词模板（迁移到新的智能体提示词）
- 旧的API接口（迁移到新的智能体接口）

### 1. 智能体封装层

**目标**：将现有模块封装为智能体，保留所有功能

**实现方式**：
```python
class AnalyzerAgent(BaseXiaonaComponent):
    """分析者智能体 - 封装AutoSpecDrafter的发明理解功能"""

    def __init__(self, agent_id: str, config: dict):
        super().__init__(agent_id, config)
        # 封装现有模块
        self.autospec_drafter = AutoSpecDrafter()

    async def execute(self, context: AgentExecutionContext) -> AgentExecutionResult:
        # 调用现有模块
        invention_understanding = await self.autospec_drafter._understand_invention(
            disclosure_file=context.inputs["disclosure_file"]
        )

        # 返回双格式输出
        return AgentExecutionResult(
            agent_id=self.agent_id,
            status=AgentStatus.COMPLETED,
            output_data={
                "structured_data": invention_understanding.to_dict(),
                "markdown_text": self._format_understanding_as_markdown(invention_understanding)
            }
        )
```

### 3. 回退机制

**目标**：当新系统失败时，自动回退到旧系统

**实现方式**：
```python
class HybridWorkflowExecutor:
    """混合工作流执行器 - 支持新旧系统切换"""

    def __init__(self):
        self.old_system = OldWorkflowSystem()
        self.new_system = XiaonuoOrchestrator()
        self.fallback_enabled = True

    async def execute(self, user_input: str, use_new: bool = True):
        """执行工作流，支持回退"""
        try:
            if use_new:
                # 尝试使用新系统
                result = await self.new_system.process(user_input)
                return result
            else:
                # 使用旧系统
                result = await self.old_system.execute(user_input)
                return result
        except Exception as e:
            logger.error(f"新系统执行失败: {e}")
            if self.fallback_enabled:
                # 回退到旧系统
                logger.info("回退到旧系统")
                result = await self.old_system.execute(user_input)
                return result
            else:
                raise
```
