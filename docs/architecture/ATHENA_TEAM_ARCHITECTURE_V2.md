## 📋 文档概述

本文档描述Athena平台的完整架构设计，包括从"小娜单智能体"到"小诺+Athena团队"的架构转型。

---

## 🎯 核心架构原则

### 设计理念

1. **专业化分工**：将小娜拆分为多个专业智能体，每个智能体专注于特定领域
2. **人机协同**：用户作为决策者，智能体作为执行者
3. **先规划再执行**：小诺0号原则，执行前必须展示计划并获得确认
4. **质量优先**：专利法律业务以质量为最高原则
5. **渐进式演进**：分3个阶段逐步实现，降低风险

## 🔄 架构转型对比

### 转型前（旧架构）

```
用户 → 小娜（大而全智能体） → 处理所有任务
```

**特点**：
- ✅ 简单直接
- ⚠️ 职责不清
- ⚠️ 难以维护
- ⚠️ 无法扩展

---

### 转型后（新架构）

```
用户 → 小诺（主智能体/编排者） → 调度Athena团队 → 处理任务
```

**特点**：
- ✅ 职责清晰
- ✅ 易于维护
- ✅ 可扩展
- ✅ 专业化

## 🎯 核心设计原则

### 1. 先规划再执行（小诺0号原则）

**定义**：任何复杂任务在执行前必须先制定执行计划，并向用户展示，获得确认后才能执行。

**实施方式**：
- 场景识别后，立即制定执行计划
- 展示执行模式、步骤、调用的智能体、预估时间
- 用户确认后执行
- 执行过程中允许用户打断和调整

---

### 2. 人机协同原则

**定义**：用户是执行复杂任务的决策者，智能体是执行者和建议者。

**实施方式**：
- 关键节点必须用户确认
- 允许用户随时打断
- 允许用户高度自由的调整计划
- 智能体提供建议，用户做决策

---

### 3. 质量优先原则

**定义**：专利和法律业务以质量为最高原则，性能和成本服从于质量。

**实施方式**：
- 不追求速度而牺牲准确性
- 不追求低成本而影响专业性
- 确保输出的专业性和准确性
- 完整的双格式输出（JSON + Markdown）

---

### 4. 渐进式演进原则

**定义**：分3个阶段逐步实现，每个阶段都可以独立运行和验证。

**实施方式**：
- Phase 1：4个基础智能体
- Phase 2：+2个审查智能体
- Phase 3：+1个无效宣告智能体
