# 专利数据导入优化方案

## 📊 数据规模分析

- **现有数据**: 2800万条记录
- **待导入数据**: 5700万条记录
- **数据格式**: CSV文件，90GB
- **字段数量**: 33-34个字段
- **预计导入后**: 8500万条记录

## 🚨 发现的问题

1. **无效数据**: 当前数据库中有1087万条记录的公开号为空
2. **重复风险**: 需要通过申请号/公开号/授权号进行去重
3. **性能挑战**: 90GB数据导入需要优化策略

## 🛠️ 优化导入策略

### 1. 预处理阶段
- ✅ 清理现有无效数据（删除公开号为空的记录）
- ✅ 创建批量导入脚本，支持并发处理
- ✅ 实现基于关键字的去重机制

### 2. 导入优化措施
- **批量插入**: 每批10,000条记录
- **去重检查**: 基于申请号、公开号、授权号
- **错误处理**: 完整的日志记录和错误恢复
- **进度跟踪**: 实时显示导入进度和性能指标

### 3. 性能优化
- **内存优化**: 分批读取，避免内存溢出
- **事务管理**: 自动提交，避免大事务
- **索引策略**: 导入完成后统一更新索引

## 📋 执行步骤

### 第一步：清理现有数据
```bash
# 删除无效记录（约1087万条）
psql -h localhost -U postgres -d patent_db -c "
DELETE FROM patents
WHERE publication_number = '' OR publication_number IS NULL;
"
```

### 第二步：执行批量导入
```bash
# 使用优化脚本导入
python3 scripts/import_patent_data_optimized.py /path/to/csv/directory
```

### 第三步：索引优化
```bash

psql -h localhost -U postgres -d patent_db -c "ANALYZE patents;"
```
