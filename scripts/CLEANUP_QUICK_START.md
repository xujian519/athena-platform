## 🔄 回滚操作

### 何时需要回滚？

- 误删了重要文件
- 清理后系统出现问题
- 需要恢复被删除的配置

### 回滚步骤

```bash
# 1. 查看可用备份
cd /Users/xujian/Athena工作平台
ls -lt .cleanup_backup_*

# 2. 执行回滚脚本
./scripts/rollback_cleanup.sh

# 3. 选择备份编号
# 例如：选择 [0]

# 4. 确认回滚
# 输入 y 确认

# 5. 等待恢复完成
```

### 手动回滚

如果脚本无法使用，可以手动恢复：

```bash
# 1. 解压备份
tar -xzf .cleanup_backup_20260420_143022.tar.gz

# 2. 查看备份内容
ls -R .cleanup_backup_20260420_143022/

# 3. 手动复制文件
cp -r .cleanup_backup_20260420_143022/path/to/file /target/path/
```

## ⚙️ 配置文件使用

### 配置文件结构

```yaml
global:
  project_root: ""
  backup_retention_days: 30
  compress_backup: true

cleanup_rules:
  backup_files:
    enabled: true
    priority: 1
    patterns:
      - "**/*.bak"
```

### 常用配置修改

**1. 调整测试报告保留时间**
```yaml
test_reports:
  retention_days: 30  # 默认90天改为30天
```

**2. 启用日志文件清理**
```yaml
log_files:
  enabled: true  # 默认false改为true
  retention_days: 7
```

**3. 禁用Python缓存清理**
```yaml
python_cache:
  enabled: false  # 默认true改为false
```

## 🔧 故障排查速查表

| 问题 | 解决方案 |
|-----|---------|
| 权限错误 | `chmod +x scripts/*.sh` |
| 找不到备份 | `ls -la .cleanup_backup_*` |
| 进程运行中 | 先`kill <PID>`，再清理 |
| Git状态异常 | 运行脚本时选择"修复Git状态" |
| 备份损坏 | 使用Git历史恢复文件 |
| 清理后系统异常 | 使用`rollback_cleanup.sh`恢复 |

## 📝 最佳实践

### ✅ DO（应该做的）

1. **定期清理**：每月至少执行一次完整清理
2. **保留备份**：至少保留一周，确认无问题后再删除
3. **检查日志**：清理后查看日志文件，确认操作正确
4. **测试系统**：清理后测试关键功能，确保系统正常
5. **更新.gitignore**：添加规则防止垃圾文件被提交
6. **监控磁盘**：定期检查项目大小，及时清理

### ❌ DON'T（不应该做的）

1. **不要跳过备份**：快速模式无备份，谨慎使用
2. **不要忽略警告**：进程运行中的警告要特别注意
3. **不要删除关键文件**：.env, config.yaml等配置文件
4. **不要强制清理**：遇到错误先排查，不要盲目继续
5. **不要删除备份**：确认系统正常前不要删除备份文件
6. **不要在生产环境测试**：先在测试环境验证清理脚本
